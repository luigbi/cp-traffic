Traffic.ini

The Traffic.ini file is used by the Traffic system to determine where various modules are located or to define the path for different features. The Traffic.ini file is typically placed in the Data folder and is referenced by all the shortcuts that are used to access the Traffic system.


There is a sample Traffic.ini in the CSI\Setup\SampleINI folder.


The following table describes each field on the Traffic.ini. Most are required. Optional fields have been marked as “Optional” in the Description.

This is the location in which archived information is saved and where the Archive retriever program looks to retrieve spot information. 

Root Drive =

This is used to define a root drive other than C:\. This is only needed if no C:\ drive exists.

WideOrbitStationXMLPath =

This is the path where “Station XML” export files will be saved for the Wide Orbit system when exporting an order with rep vehicles set to the “Send XML Insertion Order to Wide Orbit” setting in Vehicle Options. (This was previously “InsertionXMLPath”, which has been deprecated.) (V7.0 and higher) (Optional)

ProposalXMLPath = 

Allows the user to define a custom default destination folder for the Proposal XML process (the user can still manually browse to a non-default save location however). (V7.0 and higher) (Optional)

StationInvoiceImport =

Used for the Radio Station Invoice feature. This is the path where the Radio Station Invoice PDF files received from the stations will be saved and where the Import screen will browse to by default when starting the Radio Station Invoice Import process. (V7.1 and higher) (Optional)

MonitorID =

This setting affects the Task Monitor icons that appear at the top of the screen for monitored tasks, and the automatic emails that get sent when a task is not running. (Version 7.0 and above) (Optional)

"MonitorID = Off" - the Task Monitor menu icons are hidden, and the Task Monitor emails will not be sent.

"MonitorID = IconsOff" - the Task Monitor menu icons are hidden, and the Task Monitor emails will be sent when needed.

"MonitorID = EmailOff" - the Task Monitor menu icons are shown, and the Task Monitor emails will not be sent.

“MonitorID = Production” – the Task Monitor emails will indicate they are being generated from the production system.

SalesBrochure =

Used with Programmatic Buys to specify the path where Sales Brochure PDF files that can be associated with standard packages are stored. Example: “SalesBrochure = z:\csi\prod\PDF” (version 7.1 and above) (Optional)

LogGenExport = 

A path can be defined to have all log files that get saved directly to file saved here (excluding any vehicle that has a custom log export path defined in Vehicle Options).

UsingSockets =

Set to Y (or y) when using sockets to communicate with the web server, or N (or n). (Version 7.1 and above only)

ServerIPAddress =

Used to specify the Server IP Address if the Actian database is on another PC than the one being used to run the Act 1 Special Traffic System Research import. This should only be added when that is the case and an "unable to connect to database" error is occurring when running the Act 1 Special Traffic System Research import. (Version 8.1 and above only)

Establishing DDE sessions.  Different machines could require different delay loops.  Must be defined.  Enter 30.

Test System Information – Traffic.ini

This section describes the fields on the Traffic.ini that are used with a test system when the test system is configured to use the special “test locations” section of the Traffic.ini. When configuring a test system, the test Traffic system path must include the word "test" in it somewhere. For example, d:\csi\test\. This is what tells the system to get the paths from the "Test Locations" section of the Traffic.ini, and to not use the regular "Locations" section. The start in of the test Traffic system shortcut must be pointing to the location of the test traffic.ini, which also must be in a folder that has the word "test" somewhere in the path. For example, place the traffic.ini in the test data folder and set the Traffic shortcut start in path to: d\csi\test\data. The test locations section of the Traffic.ini must be filled out correctly. When configured correctly, when launching the test system, the wallpaper will show the word “test system” to make it clear that it’s the test system that’s being used.

[TestLocations]

Test system paths - same keywords as in [Locations]

Database = 

Path containing TEST *.BTR files (CSI\Test\DATA).  

It is crucial that this path is the location of the TEST databases. The rest of the paths can be Prod or Test.

ServerDatabase =

Path where database resides on the server (CSI\Test\DATA).  

Name = 

Test database DSN name.

Reports =

Path containing *.RPT files (CSI\Test\REPORTS).

Save Reports =

Path used to store reports saved by Crystal reports (CSI\Test\EXPORT).

Import =

Path used to store import files (i.e. Research data ) (CSI\Test\IMPORT).  

Export =

Path used to store export files (Bulk Feed, Commercial Changes, etc.) (CSI\Test\EXPORT).  

Logo =

Path containing Network logo for Proposal/Contract screen (CUSTLOGO.BMP) and for reports (RPTLOGO.BMP) (CSI\Test\DATA).

Help =

Path containing CSI Help manual (CSI\Test\HELP).

Exe =

Path containing CSI programs (CSI\Test\EXE).

DDF = 

Path containing data dictionary, normally this is the same as the Database path.

"MonitorID = EmailOff" - the Task Monitor menu icons are shown, and the Task Monitor emails will not be sent.

Sales Force Section of Traffic.ini

Sales Force has option to export the files to a network FTP location. By editing the FTP Info section in the Traffic.ini, the Sales Force export files can be automatically to an offsite location. If this section is not filled out, the files must manually be sent to Sales Force.

 

[FTP Info]

Test system paths - same keywords as in [Locations]

FTPIsOn =

This can be a 1 for Yes or 0 for No. If using FTP for Sales Force, set to 1.

FTPAddress =

This is the FTP address, Ex: XXX.XX.XX.XX  

FTPPort =

This FTP Port is defined by the client. Ex: XX

FTPImportDir =

User FTP import directory. This is the location where files will be uploaded. If omitted, defaults to ftp root.

LocalExports =

This is the local export folder where the Sales Force files have been placed. Ex: X:CSI\Prod\Export

FTPUID =

User’s ID to FTP location

FTPPWD =

User’s password to FTP location.
