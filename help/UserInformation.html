User Information

To create a new user, select New from the drop down and press Tab. The following section explains each field on the Name Information part of the screen.



  • Active / Dormant: This setting determines whether a user is currently in use. Dormant users can no longer sign onto the system, but will still appear on pertinent reports. Users can be made dormant or reactivated at any time
  • Sign on Name: This is the username used to enter Counterpoint
  • Name on Report: This will appear on various reports. If left blank, the Sign on Name will appear on reports including the User Activity Log
  • Password: Leave this blank when setting up a new user. When the user logs onto the system for the first time, s/he will be prompted to choose their password. This password will update the User Screen, and will remain active for the number of days defined in the Site Options -> General tab “# of days to retain password”
  • Erase Password: This will erase the user’s password, forcing them to enter a new password upon login
  • Phone: The user’s phone number can be entered here
  • City: The user’s city can be entered here
  • Email Address: When sending emails from the system, such as the Proposal XML email, this is the return email address the system will use
  • Group Number: Sales Groups are a way to allow users with the same group number but that are not on the actual proposal to share in the viewing and/or access rights of proposals on the contract screen. Note: The Group # does not extend to reporting information, it only affects what contracts can be viewed and edited on the contract screen. The Group Number rules are as follows:
    • Each user can only belong to one group (or no group)
    • If a user is defined as a salesperson on the proposal (primary or non-primary), they always get to see the proposal, even if they have a different group number than the primary or non-primary salespeople on the contract.
    • If the logged in user is linked to a salesperson user account on the User Options screen, and that salesperson user account is set with the job title of manager, planner, or negotiator on the salesperson list screen, then they can see proposals from other users with the same group number
      • If the logged in user is linked to a salesperson on the User Options screen, and that salesperson is set with the job title of "salesperson" on the salesperson list screen, then they cannot see proposals from other users with the same group number unless they are defined as a salesperson on the proposal.
    • If the logged in user is linked to a salesperson user account on the User Options screen, and the Group Number is not defined, then they can only see contracts that they are defined as a salesperson on.
    • If a user account is not linked to a salesperson, if they don't have a Group Number defined, then they can see all contracts, but if they do have a Group Number defined, then they can only see contracts from other people with the same Group Number.
  • Hub: this option only appears on Hub-Based systems, and is used to define what specific hub a user – most likely a salesperson – is allowed to access, which determines the list of vehicles they can use on the Proposal screen vehicle list.
  • Report Set: With 500+ reports in the system, reports are organized by department so that most users will not see all the reports, just the ones they may have to run. Select the appropriate report set from the report set dropdown for each user. (The reports within the different report sets can be edited as explained later in this document.)
  • Vehicle Access: This is used for station groups, to allow the Station Managers the report access they require. This is done by entering their specific station in the Vehicle Access area. For Salespeople that are defined as Manager, when you define their Station/Market as the Vehicle Access it allows that user to look at any salesperson’s paperwork in their office and change the status to Completed Proposal. The specific reports that limit reporting based on the Vehicle Access are:
    • Ageing by Vehicle        
    • Ageing by Salesperson
    • Affidavit of Performance 
    • Billed and Booked Recap        
    • Billed and Booked 
    • Billed and Booked Commissions        
    • Missed Spot Report
    • NTR Billed and Booked
    • Sales Commission Report        
    • Spots by Advertiser
    • Weekly Sales Activity by Month
  • Default Vehicle: If a default vehicle is selected, this will be the first vehicle that appears in the schedule lines in the Proposal/Order Screen
  • Salesperson: When a user is defined as a salesperson, there are certain system rules that are applied to their system access. See the Group Number definition above for additional information on how this setting works with Group Numbers for viewing/editing contracts on the contract screen. Sales assistants, sales managers, and planner’s ability to view and enter proposal/contracts are affected by the settings defined for them in Lists -> Salesperson. The salesperson job title allows specific access based on the title defined in the Salesperson Screen:


Salesperson
  • Salespeople do not have the ability to select printable orders
  • The Billing button to allocate promotion or merchandising on the Proposal Screen is disallowed
  • The new Advertiser/Agency screen is modified to require the minimum answer and the accounting questions are skipped
  • When using Projections, they can only enter or modify their own projections
  • The option to use or modify Reservation orders is not shown

Negotiator/Planner

Can do all things that salespeople can do, plus extra capabilities available only to Planners

  • Merchandising
  • Promotion revenue
  • Projections

Manager

Can do all things that salespeople and planners can do plus extra capabilities 

  • Approving completed proposals
  • View certain reports
