Export Inventory and Spots

Export Inventory and Spots is a standalone export program that creates a Spot Export file and an Inventory Export file, that can be imported into a Hyperion database. No files are updated during this data gathering process and it can be run at any time.


The two files are created each time it is run for default dates of today minus three days to the last date a spot is scheduled or user entered dates. If the export is rerun for the same dates, it will write over the previous two export files.

Export Options

Manual

The manual export allows you to define the start and end dates for spot and inventory for all vehicles.  


Create a shortcut to run the Export Inventory and Spots program.


Target:  X:\csi\prod\exe\exportinvspots.exe


To allow the user to define dates add

“ /UserInput” to the target line


Start In:  X:\csi\prod\data 

(location of the Traffic.ini file) 


When you run the application you will be able to enter a start date and/or end date.  


Start date will default to the current date minus 3 days.


End date will be the last spot booked in the future.


Contract number is only used for verification testing.


Change the dates or verify the dates, and select the Export button.

Automated

The export can be defined to run automatically based on the user day and times to run the export, ideally at night. The start date will default to the current date minus 3 days. The end date will be the last spot booked in the future.

Option 1: CSI Server Manager  

Add entry to csi_server.ini: 


[ExportInvSpots]

Enabled = 1

WeekDays = 1111111

StartTime = 2:00am

Interval = 86400

Exe = x:\csi\prod\exe\exportinvspots.exe

StartInFolder = x:\csi\prod\data

CommandLine = 

KeepProgramRunning = 0


Refer to the help document “Ini Values and Examples” on the Counterpoint website for additional CSI Server.ini set up information 

Option 2: Windows Task Scheduler

  1. Go to Windows Control Panel, select “Administrative Tools”, and choose “Task Scheduler”.
  2. Select Create Task.
  3. Enter a name for the task on the General tab.
  4. Select the option to run it whether the user is logged in or not.
  5. Press the Trigger tab. Enter the schedule on which you wish the task to run (for example, weekly) and the start time.
  6. Press the Actions tab. In the Program field, browse to the exportinvspots.exe file located in the exe folder.
  7. Enter the location of the traffic.ini file in the Start in field.
  8. Check the settings on the Conditions and Settings tab, then press Okay.
  9. The export will now run as a scheduled task on the schedule you defined.

Export Verification 

The ExportInvSpots.txt shows you a list of each vehicle and indicates the status or stage of the export what was processed during the export. The file is stored in the x:\csi\prod\data\messages folder.  


The Export message results are appended to the one file. This file should be periodically deleted to prevent it from getting too large and difficult to read.  

Spot and Inventory File Names

The files are stored in the x:\csi\prod\export folder or wherever your traffic.ini export setting is defined.

Inventory File

The inventory file is Inv-mmddyy-mmddyy.txt. The two dates are the start and end date that were entered or calculated automatically.

Spot File

The spot file name is Spot-mmddyy-mmddyy.txt. The two dates are the start and end date that was entered or calculated automatically.

Spot Data Field Descriptions

Spot Time

Each spot has two time fields – scheduled and aired.


Scheduled time - Spots contain the time that they are (or were) scheduled to air.  If they are pre-empted, and have never been scheduled, the scheduled time is the start time of the sold daypart.  If they have been previously scheduled, and then pre-empted, the scheduled time is the last avail time they were scheduled.


Aired time – If in the future, the aired time is always set to whatever the scheduled time is.  If in the past, the aired time is set to whenever the spot actually aired.  If the spot did not air, the aired time is set to whatever the scheduled time is.

Spot Status

Miss: a spot that will not or did not aired, either because there is no room, too many of that type of product, or removed on purpose.

Makegood: a spot that takes the place of a Missed spot, and will be referenced back to original missed spot on the invoice.

Outside: a spot that takes the place of a Missed spot, but will be shown on the invoice as a regular spot.

Cancel: a spot that was missed but will not be made good, usually at the request of the advertiser.

Hidden: a spot that was missed but will not be made good, and should not appear on the invoice.

Regular: a spot not one of the above.

Spot Type

Open billboard: a spot at the start of a program announcing who the sponsor will be.

Close billboard: a spot at the end of a program announcing who the sponsor was.

Fill: a free spot generated by the system to fill avails that need to be filled and were not sold, or a spot given to an advertiser in an unsold avail for goodwill purposes.

Regular: a spot not one of the above.

Contract Type

PSA and Promo type contracts are automatically ignored.  

Reservation: a spot booked to temporarily hold space until the advertiser makes up his mind.

Remnant: a spot sold at the last minute at fire sale prices.

Direct Response: a spot sold which will be paid for based on the number of calls made by the public in response to the ad.

Per Inquiry: a spot sold which will be paid for based on the number of calls made by the public in response to the ad.

Regular: a spot not one of the above.

Vehicle Group Code

A Vehicle Group is an identifier to group vehicles together for reporting purposes. Each vehicle can be assigned a value in each of five user defined groups. 


Market is the area that this vehicle is reaching. If you have the ‘market’ groups on, you will be asked which group you want to use on the Proposal, Order, Spot and Log screens.


Research is Rated, Non-Rated or Suburban.


Sub company is typically a way to group together vehicles whose accounting revenue

should be reported together.


Format is the type of vehicle it is. Comedy, AOR, etc.


Subtotals are a further way of classifying together a group of vehicles for reporting purposes.
