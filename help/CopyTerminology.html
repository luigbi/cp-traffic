Copy Terminology

This section defines the basic Copy terms that will be used throughout this document.


  • Assign: Assigning copy is the process in which a specific piece of copy inventory is attached to a spot, following the copy rotation instruction rules that were entered, so that the copy will appear with the spot on the log and any log spot export such as the Counterpoint Affidavit export. The primary way to assign copy is by generating the log with the “assign copy” checkbox checked on. When the system assigns copy, it examines the most recently entered copy rotation first, and if its restrictions fit the spot, then it gets used. If not, the next oldest copy rotation instruction is tested. This process continues until one is found that meets the spot criteria. If no rotations match the spot criteria, no copy is assigned to that spot. Such potential issues can be found prior to generating the log by running and analyzing the Contracts Missing Copy report to find what spots if any are missing copy. If any changes are made to copy after the log has been generated, the log must be regenerated and copy must be assigned again. (It is also possible to manually assign copy on the Copy screen, although the best practice is to assign copy at the time of generating logs.)
  • Blackout: Blackout copy is copy that airs and replaces the copy for one advertiser with copy from another advertiser. Blackouts are used with Copy Regions and work similarly to Split Copy or Regional Copy. The difference is that Regional or Split Copy use the same advertiser and Blackouts replace the advertiser with copy from a different advertiser.
  • Cart: Cart numbers are internally assigned numbers for copy inventory. Cart numbers are generally used for automation systems. Carts can be shown on logs, and can be used along with or in place of ISCI codes.
  • Copy Inventory: Copy Inventory is an actual piece of copy (ISCI and/or Cart). Inventory is tied to an advertiser and product, has a designated length, and may have a Creative Title attached, and possibly other information as well, such as a live copy script that will be read live on air. Inventory is referenced in copy rotation instructions. Once inventory is entered, it will remain in the system until purged (if applicable). 
  • Copy Region: A Copy Region is a group of stations with a commonality that the advertiser is trying to target with unique copy. The Region can be based on six possible regional categories: DMA market, Format, MSA market, State, Station, and Time Zone. For example, there could be a “Spanish Only” region that is made up of stations with the “Spanish” format, or a “California” region that consists of stations defined with the state of  “California”. One advertising campaign can include multiple Copy Regions based on advertiser/agency criteria.
  • Copy Rotation Instructions: The Copy Rotation Instructions describe what inventory is to air and the valid vehicles, dates, times and days that the inventory may air. Ratios may be entered that specify the percentage that each piece of copy will air to the total (for example, Copy A should air twice for every time Copy B airs once). 
  • Generic Copy: Generic copy is the copy that airs on all stations not airing regional or blackout copy. If no regional or blackout copy is used, then all copy is in effect “generic”. The distinction is only important when regional and blackout copy is used as when using regional and blackout copy, the generic rotation must always be entered first, then the regional and blackout copy. This is done to ensure that any station that does not meet the regional criteria will still get the correct generic copy and that any stations that do meet the regional criteria will get the correct regional or blackout copy.  
  • ISCI: An ISCI code is a unique code used to refer to a specific piece of copy. The ISCI code is typically provided by the agency.
  • Regional Copy or Split Copy: Regional Copy or Split Copy is copy that is meant to air only on those stations that meet the region criteria defined for a copy region that is used for a specific set of rotation instructions. Typically a single advertiser will have one set of copy airing in one region, another set of copy airing in another region, and so on.
