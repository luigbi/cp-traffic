Podcast and Ad Servers

With software version 8.1, Counterpoint has added a large number of features specifically for podcasts and ad servers. This document is a complete overview of the Traffic system features used by podcasts and ad servers.

Key Features

  • Counterpoint supports selling podcasts by spots or by impressions for Ad Servers such as Megaphone and AdsWizz.
  • A single contract in Counterpoint can be used for both air-time lines, podcast spots, and impressions based buys sold by CPM or for a flat rate.
  • Digital lines for dynamically inserted ads are entered on the Digital tab on the Proposal and Orders screen, using the vehicle, ad location, date range, impressions goal, and CPM or flat rate cost.
  • Ordered impressions are shown on the proposals/contracts report for Digital lines.
  • The Ad Server Billing Discrepancy export can be run for a month and year to check the number of impressions delivered, and to check for any lines active for the month with no impressions delivered against them.
  • Impressions delivered by Ad Servers can be updated in Counterpoint, and appear on the invoice.
  • Digital related reports appear in the Digital report category in the Traffic system report list.

Ad Server Overview

Ad Servers are used to dynamically insert ads into podcasts or other media, so that different listeners listening to the same audio may hear different ads. Ad Server ads are sold by impressions by CPM or for a flat rate. Because the ads are dynamically inserted, ad server buys allow for special targeting rules, such as device targeting, geographic targeting, etc. 

Ad Server Workflow

  • Contracts with digital line items will be entered, saved, and scheduled in Counterpoint.
  • In the future, during scheduling, Counterpoint will use the Ad Server API to send the lines to the Ad Server. If the Ad Server API is not available, after a proposal becomes an order in Counterpoint, the digital lines should be manually entered in the Ad Server program.
  • Ad Server users will traffic and manage the podcasts in the Ad Server program, and the Ad Server will track impressions achieved as listeners listen to the audio.
  • In the future, Counterpoint will directly connect to the Ad Server API to get the impressions delivered for each line item for the month. If the Ad Server API is not supported or not available, the Ad Server Posting Screen in Counterpoint will be used to manually post the impressions delivered by the Ad Server in Counterpoint.
  • When invoices are run, digital lines will show the impressions delivered for the month, and calculate the monthly total for each line using the CPM (for CPM lines), or the monthly cost (for Flat Rate lines).

Podcast Spot Overview

Podcasts can also use spots, where the ad is “baked-in” as part of the podcast audio file, so all listeners hear the same ad. Podcast spots are not sent to an Ad Server, and do not allow for special targeting rules like digital lines do, as they are not dynamically inserted. In Counterpoint, a vehicle that uses podcast spots works just like an air time vehicle, with programming libraries, inventory, and spots that get managed in Counterpoint.

Podcast Spot Workflow

  • Contracts with podcast spot lines will be entered, saved, and scheduled in Counterpoint.
  • During scheduling, the spots will be created in Counterpoint, at which point they can be trafficked on the Spots screen, just like air time spots.
  • Logs can be generated for podcast spot vehicles.
  • Podcast spot vehicles can be configured to show “week of” dates instead of specific air dates on the invoice.
