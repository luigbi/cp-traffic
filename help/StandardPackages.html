Standard Packages

Vehicles can be grouped, or bundled together in Real or Virtual packages. The vehicles that make up the package are called ‘hidden lines’ and they can be included or excluded from the printed contract. The package schedule line is independent of the hidden lines of which it is comprised. 


Whether Real, Virtual, or both package types are used is determined by a question in Traffic Site Options->Sales: “Packages Allowed: Virtual—Real”. 


Packages are frequently used to bundle unsold inventory, increase ratings, or target a particular demographic. 

Package Characteristics

There are two types of packages, Real and Virtual. Each package type has unique characteristics

Virtual Packages

Virtual packages allow the user to define any number of spots on the package line to be grouped as one or more virtual spots, completely independent of the number of spots that make up the package. These "virtual spots" are intended to represent the actual spots that air, and are shown on invoices with representative, not actual, times. Since the invoiced times are representative times, Virtual Packaged buys always invoice as ordered, not as aired, regardless of the billing setting in Site Options->Invoicing.

Real Packages

Real packages group any number of spots together and show the true number of spots to be aired. On invoices, the actual hidden vehicles, dates aired, and times aired will appear.

Standard Package Screen

To view, edit or create Standard Packages, click on the “Std Pkg” button on the Rate Card screen to bring up the Standard Package screen. 


Vehicles and dayparts shown on the Standard Package screen use the color coding listed below.


Vehicles

Red: Dormant

Blue: Package vehicle


Dayparts

Red: Dormant


To create a new Standard Package, first select “New” from the Package dropdown in the top center of the screen, fill in the header information and then select each vehicle/daypart you wish the package to include by entering a number of spots in the package in the “# Spots” column.

The following list describes the different fields available in the header area.

  • Standard Package Name: This is the Package Name that will appear on your contracts and Invoices.
  • Price: This determines how the Package Price will be calculated; click on the toggle to change the calculation method.
    • Rate: The package price will be proportionally distributed to the hidden lines based on the rate defined for each hidden line vehicle/daypart on the Rate Card.
    • Audience: The package price will distribute the average spot price to the hidden vehicles based on the audience share per hidden vehicle for the demo category.
    • Percent: The package price will be distributed to the hidden vehicles based on the percentages instead of spot counts, entered for the hidden vehicles. The percentages entered must equal 100%.
    • Spot Count: This will distribute the dollars based on each hidden vehicles spot count, in relation to the total dollars.
  • Package Type: There are two types of packages: Virtual and Real, although some systems may only have one type enabled. Each package type has unique characteristics.
    • Virtual: Package spot counts are independent of the total spot count that makes up the package. Virtual spots and times will appear on the invoice
    • Real: Package spot counts equal the number of spots that make up the package. The actual hidden vehicles and true air times/dates will appear on the invoice
  • Programmatic: The Programmatic field is a toggle field that can be toggled between yes and no. This option is only available on version 7.1 and above. If the Programmatic feature is not enabled in Traffic Site Options, then this field cannot be clicked on and used. (This is part of a future enhancement.)
  • Sales Brochure: The Sales Brochure field allows you to select a PDF file to associate with the standard package, for use with the Programmatic Buy feature. The folder where the PDF files are stored is either pulled from the “SalesBrochure” setting on the Traffic.ini, or from the standard Import folder if the “SalesBrochure” definition is not entered on the Traffic.ini. This option is only available on version 7.1 and above. If the Programmatic feature is not enabled in Traffic Site Options, then this field cannot be clicked on and used.
  • Alter Name: When set to No, it prevents users from editing the package name or selecting a new vehicle for the package once the package has been created on the Proposals/Orders screen. If set to Yes, the package name can be edited or the package vehicle can be switched to another vehicle once the package has been created.
  • Alter Hidden: When set to “Yes”, this indicates the hidden lines spots from this package can be modified on the proposal/order screen. When set to “No”, they can’t be modified. When set to “Comment/Audio only” (“Cmmt/Audio only”), the spots can’t be modified but the line comment and audio type can be added and edited. When set to "Rates Only" (version 8.1 and above only), then the hidden line rates and the comment can be edited.
  • Demo: This is the demo used to calculate the package price, if applicable.
  • Market: If applicable, this is the Market that will be associated with this package in the Proposal/Order Screen. This package will only be accessible for the Market selected.
  • Index: The index allows you to increase or decrease the package spot cost when the system computes the average spot cost for the package.
  • Act 1 Code and Settings: if a standard package is configured with the “alter name” setting of “no”, and the “alter hidden” setting of “no” or “comment/audio only”, or "rates only", then a default Act 1 Lineup code and default settings can be entered for that standard package in these fields, and when that package gets added to a proposal, the default Act 1 Code and Settings will be automatically added to the package line (version 8.1 and above only).

Proof Button

To create a report that can be used to double check the vehicles and dayparts that make up a standard package, first select the previously created standard package from the package dropdown, then press the Proof button at the bottom of the Standard Package screen to generate the report. It creates a comma delimited file that lists the vehicle names and dayparts for the selected standard package. The file gets created in the Exports folder as specified on the Traffic.ini. The filename of the report will match the name of the package.

Adding a Standard Package to the Rate Card

To add a standard package to the rate card, select the standard package from the vehicle field of the first available line on the rate card then press Tab. This will bring up a question that reads “Computer package price”.


  • “Yes” will get all the hidden line prices for this Standard Package and will multiply the total by the index value to arrive at the package price value. 
  • “No” will bypass entering a rate.  You can enter your own price if you choose.


Next define the daypart, then tab through to the next line. Press Save once all the required changes have been made to save your changes.
