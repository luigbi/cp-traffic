Wide Orbit

Variable Length, comma delimited, Filename: YYMMDD00.log (where 00 is hard-coded - see below), or YYMMDD.log by default on version 8.1 and above.


All records terminated with c/r l/f.


(YY = year, MM = month, DD = day).  


Sample Record :  20:20:00,,CA,DA9862,”Lifetime channel/Sunday-New Series”,”:131259:Stn”,00:30,

Note: Wide Orbit cannot process a filename other than “YYMMDD”, therefore multiple vehicle exports for the same day must be placed in different folders. To set up Wide Orbit export, create two entries in the Traffic.ini Locations section. By having these folders defined, that’s where the default path will take the user:


       WideOrbitImport = x:\csi\v58\WideOrbit import  (full path to wide orbit import folder)

       WideOrbitExport = x:\csi\v58\WideOrbit export  (full path to wide orbit export folder)


For each vehicle to export, go to the Vehicle Options screen -> Log tab. Enter the full path where the export file will reside. The subfolder name should be only the vehicle’s station code.

"00" in Filename

On version 8.1 and above, by default, it will now exclude “00” from the filename. By default, the filename format is now YYMMDD.log. However, if the “00” is required in the filename, a new “Append 00 to filename” checkbox has been added to the Automation export screen when the Wide Orbit automation export is selected. This checkbox is not selected by default. If the Wide Orbit export gets run with the “Append 00 to filename” checkbox checked on, then 00 will be appended to the filename like how it worked before.
