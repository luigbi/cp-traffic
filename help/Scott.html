Set Up

Traffic.INI

Edit the Traffic.INI in the CSI\Prod\Data Folder by adding a new line to the Locations section:


ScottExport = x:\CSI\PROD\Export\Scott

Export Folder

By having multiple folders defined, you can run multiple Scott Exports for multiple vehicles, and easily identify them.


Create a new folder in the CSI\Prod\Export Folder called Scott

The export folder for “Scott” needs to be the same name you entered in the Traffic.ini lines above.


Within the Scott folder, make a folder for each vehicle that will be using the Scott Automation Export, for example: KCCCFM, KDDDFM, etc.

Vehicle Options

In the Vehicle -> Options -> Log tab define an automation export path for each vehicle using Scott. This path should match the sub-folder names defined in the CSI\Prod\Export\Scott folder.

Setup

Traffic.INI

Scott V5 cannot process a filename in a format other than “YYMMDD”, therefore multiple vehicle exports for the same day must be placed in different folders.  To set up Scott V5 export, create two entries in the Traffic.ini.  By having these folders defined, that’s where the default path will take the user:

ScottImport = x:\csi\prod\scott import  (full path to the Scott import folder)

ScottExport = x:\csi\prod\Scott export  (full path to the Scott export folder)

NOTE: Scott & Scott V5 exports will use the same .ini entries.

Site Options

Turn Scott V5 on in the Site Options -> Automation Tab.

Vehicle Screen

Make sure each vehicle has a unique station code. This code will be used to store the export file.

Vehicle -> Options -> Log

For each vehicle to export, enter the full path were the export file will reside in the Vehicle -> Options -> Log screen, “automation export drive/path”.
