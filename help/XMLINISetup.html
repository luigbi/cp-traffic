XML.INI Setup

  1. Create an XML.INI file for your X-Digital Interface.  A sample file can be found in the x:\csi\prod\setup\SampleINI folder.
  2. The XML.INI file must be placed in the same folder where the Affiliat.ini is placed.  
  3. You need to determine if you will export to one X-Digital head end provider or if you need to export to two different X-Digital head end providers.  

Single Head End Provider

[XDigital] 

This is the header section of the single head end XML

Host =

This is the IP address of the server.  X-Digital will supply this information to you.  

Example:  207.114.205.20

Port =

Port Example:  80

WebServiceURL =

X-Digital will supply the URL of the server and service to you.

Example:  /svc/XYZdService.asmx

DomainNameURL =

X-Digital supplies this information. Example:  http://xnms.xdigital.com

Security Token =

X-Digital supplies this information. It is a 128 bit digital ID to prevent unauthorized transmissions to the server.

XML Version =

This is always 1.0 unless otherwise specified by X-Digital.  Example:  1.0

XML Encoding =

This is always “utf-8” unless specified otherwise by X-Digital.  Example:  utf-8.

SoapEnvolope =

This defines the SOAP header that surrounds the XML data. It should always be entered as shown unless specified otherwise by X-Digital.  

Example:  xmlns:soap='http://schemas.xmlsoap.org/soap/envelope/' xmlns:xsi='http://www.w3.org/2001/XMLSchema-instance' xmlns:xsd='http://www.w3.org/2001/XMLSchema'

SoapBody =

This is always entered as soap:Body

AuthorizationChunkSize=

This groups 4,000 spot insertion records and authorizations into one ‘soap envelope’, if not changed to a different value in this field. This is optional.

LogFile = 

All error messages are captured in a txt file which is stored in the LogFile location. If the location is not defined, the log file will be stored in the EXE folder. If the location is not valid, the export will continue, but the log file will not be created. 

Change this value to match with the folder configuration

Example:  X:\csi\prod\data\messages\XDigitalLog.txt

ClearLog = 

Set it to N. This prevents the debug log from being overwritten. 

AllPorts =

If 0, the station will be not be able to change the port setting. If left out of the INI or greater than 0, all Ports will be available to the station. This is optional.

ReceiverIDSource =

This is an optional setting that determines from where the Site ID is obtained. 


A = use the Site ID defined on the agreement 

B = use the agreement Site ID first if it exists, if not, use the Station Site ID 

S = use the Station Site ID only 

N = Don't send the Site ID 

Retries =

This determines the amount of times the system will attempt to resend the export if there are any issues. If not defined, the system defaults to three. This is optional.

SharedHeadEndID =

If the X-Digital head end is a shared head end, enter the shared head end ID here. For example, SharedHeadEnd = 1. Counterpoint support maintains a list of shared head end IDs. Please contact Counterpoint support for additional information on shared head ends. If the X-Digital configuration does not use a shared head end, this line should not be added.


Single Head End Example

[XDigital]

Host = 207.114.205.20

Port = 80

WebServiceURL = /svc/XYZdService.asmx

DomainNameURL = http://xnms.xdigital.com

SecurityToken = 9452831D-C072-4E1C-8E23-689A491156BC

XMLVersion = 1.0

XMLEncoding = utf-8

SoapEnvolope = xmlns:soap='http://schemas.xmlsoap.org/soap/envelope/' 

AuthorizationChunkSize = 4000

xmlns:xsi='http://www.w3.org/2001/XMLSchema-instance' xmlns:xsd='http://www.w3.org/2001/XMLSchema'

SoapBody = soap:Body

LogFile = D:\csi\v60\Data\Messages\XDigitalLog.txt

ClearLog = N

AllPorts = 0

Retries = 3

Dual Head End XML Sections

BackOffice: This section is defined for game agreements and used to send station information 

Vantive: This section is defined for non-game agreements 

XDigital-CU: This section is defined for the use of spot insertions and programs (non-sports only) 

X-Digital-CL: This section is defined for the use of spot insertions and file delivery 

Multiple Head End Provider

  1. Define two separate [XDigital] sections as follows:
    1. [XDigital-CU] for Cumulus
    2. [XDigital-CL] for Clear Channel
  2. Add a Provider line with the name of the provider
    1. Provider = Cumulus
    2. Provider = Clear Channel
  3. This will allow you to select the correct provider from the export screen

Vantive XML.INI

If there are vehicles using the ISCI avail form and the Cumulus export, and they need to export station and/or agreement information, you must define the Vantive section in the XML.INI and activate the appropriate settings Affiliate Site Options.

 

[Vantive] 

Host =xxx.yyy.zzz.nn 

Port = 80 

WebServiceURL = /svc/VantiveService.asmx 

SecurityToken=XXXX 

XMLVersion = 1.0 

XMLEncoding = utf-8 

DomainNameURL = http://xnms.xdigital.com 

SoapEnvolope = xmlns:soap='http://schemas.xmlsoap.org/soap/envelope/' xmlns:xsi='http://www.w3.org/2001/XMLSchema-instance' xmlns:xsd='http://www.w3.org/2001/XMLSchema' 

SoapBody = soap:Body . 

LogFile = Z:\csi\prod\Data\Messages\XDigitalLog.txt 

ClearLog = N

BackOffice XML.INI

The station information gets sent using a section in the XML.ini called "BackOffice." 


The actual INI settings should replace the placeholder values below with the actual values. The Host, Port, WebServiceURL, SecurityToken, and LogFile path may all require updating.

[BackOffice]

Host = xx.xx.xx.xx

Port = 80

WebServiceURL = /svc/backoffice.asmx

SecurityToken=xxxx

XMLVersion = 1.0

XMLEncoding = utf-8

DomainNameURL = http://xnms.xdigital.com

SoapEnvolope = xmlns:soap='http://schemas.xmlsoap.org/soap/envelope/' xmlns:xsi='http://www.w3.org/2001/XMLSchema-instance' xmlns:xsd='http://www.w3.org/2001/XMLSchema'

SoapBody = soap:Body 

LogFile = x:\csi\prod\Data\Messages\XDigitalLog.txt

ClearLog = N


This section is also required for sending agreement information for agreements with non-game vehicles that are using the ISCI avail form with a dual head end configuration and by the X-Digital import.
