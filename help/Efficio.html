Efficio

There are two Efficio CRM exports: Revenue and Projections. The revenue and projections export can be generated by standard broadcast or calendar months. For standard broadcast months, the revenue export gathers receivables information from billed months, and the projection export gathers contract information from unbilled months, starting from the month after the last billed and closed month up through the last month in the future with contracts ordered for it. For Calendar month exports, both revenue and projection data comes from schedule Order lines. 

Setup

Site Options -> Options

Activate the feature in Site Options -> Options. You will a keycode from support@counterpoint.net. 

CSI_Server.INI

The exports can be setup to run automatically by adding the export names to the Section Names line on the CSI_Server.ini and by creating entries for each export, as described below.


First, add “EfficioRevenue” and/or “EfficioProjections” to the SectionNames line near the top of the CSI_Server.ini, as shown below.


Section Names example:


SectionNames = EfficioRevenue, EfficioProjections


Then add entries for both exports, as shown in the boxes below. (The lines with a semi-colon are comments that explain what each value means.) The first example is for the Efficio Revenue export, the second example is for the Efficio Projections export.

  

[EfficioRevenue]                                             

Enabled = 1

;1=Yes/On, 0=No/Off

WeekDays = 1111111        

;1=Yes/On, 0=No/Off

StartTime = 6:00AM

;the time that it should start running

Interval = 86400                

;Interval is in seconds, i.e, 86400 = 24 hours

Exe = c:\csi\prod\exe\Exports.exe

;point to the exports.exe file

StartInFolder = c:\csi\Prod\data 

;must point to the folder where the traffic.ini is located

CommandLine = Auto-Efficio Revenue            

;Needed to run a Revenue as a scheduled task

KeepProgramRunning = 0        

;1=Yes/On, 0=No/Off. Should be set to 0 for this export


[EfficioProjections]                                               

Enabled = 1                        

;1=Yes/On, 0=No/Off

WeekDays = 1111111        

;1=Yes/On, 0=No/Off

;the time that it should start running

;Interval is in seconds, i.e, 86400 = 24 hours

;point to the exports.exe file

;must point to the folder where the traffic.ini is located

;Needed to run a Projections as a scheduled task

;1=Yes/On, 0=No/Off. Should be set to 0 for this export

Exports.INI

The Exports.ini file must also be created and placed in same folder as the Traffic.INI, typically the Data folder. The csi_server.ini must be placed in the EXE folder. The Exports.ini file is used to specify the export parameters when it is automatically generated. There are two example Exports.ini files shown below, one is for use with the single export parameter setup, the other is for use with the multiple export parameter setup. In both examples, the name in brackets must match the name that appears after “Section-“ on the corresponding CommandLine on the CSI_Server.ini.


[Efficio Cal Net] 

Calendar=Cal Dollars

Net Export = c:\csi\prod\export


What is Sent

Standard Revenue

  • Revenue data is based on standard broadcast month billing.  
  • Gross, net, and T-Net dollars are listed.
    • Acquisition costs and applicable commissions are included in T-Net calculations.
  • The export defaults to the most recently billed and closed month.
    • Past months may be specified  when running the manual version.
    • Future months, or a month not yet fully invoiced, are not allowed.
  • Cash and Trade invoices are automatically included.
  • Airtime and NTR (Non-traditional revenue) invoices are automatically included.
  • Local, Regional, and National invoices are automatically included.
  • PSA/Promo invoices are automatically excluded.

Calendar Revenue

  • Revenue data is gathered from scheduled order lines.

Standard Projections

  • Projection data is based on standard broadcast month billing.  
  • All projection data is gathered from future orders. 
  • The start month is always the month following the last month billed and closed.
  • The end month will always be the last month ordered farthest into the future.
  • Cash and Trade orders are automatically included.
  • Airtime and NTR (Non-traditional revenue) orders are automatically included.
  • Local, Regional, And National orders are automatically included.
  • PSA/Promo orders are automatically excluded.

Calendar Projections

  • Revenue data is gathered from scheduled order lines.

Running the Export

You can manually run the export at any time from the Tasks -> Exports -> Efficio menu.


Select whether you wish to export revenue or projection data. 


Revenue defaults to Calendar month and to the last complete month billed. Select the billing method and any month completely billed in the past.

Projections defaults to Calendar and the month following the last month completely billed. Select the billing method and month to export.


Both exports files will default to be saved in the export folder path defined in the Traffic.INI. You may browse to a different location, should you wish to do so. 


Once the month and export folder is defined, press Export.

Test Export

If you wish to generate a test export, you can do so by exporting a single contract. Enter the contract number in the Contract Number field, and press Export. The file will save to the defined export folder.

Output

The Revenue and Projections exports both produce CSV files; the dates in the Revenue file are always in the past, while the dates in the Projection files are always in the current or future months. 

File Names

Revenue File

The Revenue data produces a file named EfficioRevCCCMmmYYYYA CCCCC.CSV.


Efficio is hard-coded

Rev indicates that this is a revenue export

CCC is either CAL (for Calendar) or STD (for Standard Broadcast) 

Mmm is the billing month, example Jan, Feb…Dec

YYYY is the billing year

A is incremented each time the export is run for the same month/year

CCCCC is the Client Name abbreviation from Site Options -> General tab

CSV indicates the file format, which is comma-separated variable.

Projections File

The Projection data produces a separate file, Efficio ProjCCCMmmYYYYA CCCCC.CSV. The naming structure is identical to the Revenue file, with Proj indicating that the file contains projected data.

File Data

The first record in the CSV file is a line of headings. 

Order#, Vehicle Name, Cash/Trade, Air Time/NTR, Local/Regional/National, Sales Source, Sales Office, Salesperson, Agency, Advertiser, Product, Order Type, Primary Competitive, Secondary Competitive, Invoice #, Year, Month, Gross Amount, Gross Split Amount, T-Net


The second line is the generation date and time.

As of 06/05/14 8:50:30AM


The data begins in the third line (this example is a revenue export).

12345,“George Carlin Show”,”C”,”A”,”L”,”Curtis Media”,”Atlanta”,”John Doe”,”Ted Bates”,”Sears”,”Batteries”,”C”,”Department Store”, “Auto Parts”,22239,2014,5,1000,1000,850


  • The lowest level of detail is a vehicle within a contract
    • There can be multiple records for the same contract, due to multiple vehicles and/or salespeople splits
    • One record for each split salesperson is generated per month, if applicable
      • The total month’s gross amount is placed in the first salesperson’s record, in addition to the splits for each salesperson on the order
      • For example, 3 salespeople share in the buy at 50%, 25% and 25%.  The month’s total gross is $1000.  Salesperson 1’s share is $500, salesperson 2 & 3’s share is $250.  In a separate column (Gross Direct), salesperson #1 will show $1000, salesperson 2 & 3 will show $0 in that column
  • Orders that are part cash/part trade generate two records, one with dollar amounts scaled to the cash portion, and the other scaled to the trade portion
  • The invoice number is 0 if projected data
  • TripleNet figures are determined by subtracting acquisition costs from the net dollars, then subtracting any merchandising/promotions 
    • Triple Net (T-Net) = Net (Gross dollars - Agency commission) – acquisition cost (spot cost (# spots x spot cost) - (# spots x acquisition cost)) – merchandising/promotions costs
  • A separate entry is shown for a Merchandising or Promotion transaction
  • The Gross field and Gross Split column will be blank, those bottom line totals are not affected
  • The TNet field will show as a negative amount, to be subtracted from totals
