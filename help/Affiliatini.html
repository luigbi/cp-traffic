Affiliat.ini

The Affiliat.ini file is used by the Affiliate system to determine where various modules are located or to define the path for different features. The Affiliat.ini file is typically placed in the Data folder and is referenced by all the shortcuts that are used to access the Affiliate system.


There is a sample Affiliat.ini in the CSI\Setup\Sample INI folder.


The following table describes each field on the Affiliat.ini. Most are required. Optional fields have been marked as “Optional” in the Description.

Affiliat.ini Field Descriptions

[Database]

Names defining system

Name =

Name of the database used when specifying ODBC System DSN (Start button -> Control Panel -> ODBC, Then select System DSN tab.  Specify database name, database path, and DDF path)

[Locations]

Paths defining system

DBPath = 

Complete path to the database

Reports =

Path containing Affiliate Crystal *.RPT files (use CSI\PROD\REPORTS)

Export =

Path used to store affiliate export files (use CSI\PROD\EXPORT). 

Import =

Path used to store affiliate imported files (use CSI\PROD\IMPORT

Exe =

Path containing CSI affiliate programs (use CSI\PROD\EXE).

Logo =

Path containing background pictures (CSI640A.Gif; CSI800A.Gif; CSI1024A.Gif; CSI1280A.Gif).

ContractPDF =

This points to the folder in which PDFs of the contracts between the affiliates and the network are stored.

LogActivityFileName =

(Optional) This entry should only be added by Counterpoint for logging purposes. This is the name of the file for which you wish to log inserts, updates, and deletions. The txt file is placed in csi\prod\data\messages, and is called logactivityfor_xxx.txt, where xxx is the file name.

  • Example: “CPTT”

Timeout =

XXX is the number of seconds to wait without getting a response back from the database to indicate that the database is not responding.  If not defined, then the standard timeout is used for Remote Data Objects (RDO) interface to ODBC.  If one needs to be entered, start with 5 minutes (300 seconds).  The affiliate error message has ‘S1T00 timeout expired’ in message.

StartInFolder =

Defines the start in path where the traffic.ini or affiliat.ini is stored.  Example: StartInFolder = X:\CSI\Prod\Data

CommandLine =

Defines the command line parameters if needed, to be passed to the application being started.        Example: CommandLine = /Debug

AudioExtension =

The value entered in this path will affect the audio extensions exported throughout Traffic and Affiliate. Affected Traffic exports: Audio MP2, ISCI Cross Reference, and Copy Bulk Feed. Affected Affiliate exports: Export Star Guide, ISCI Cross Reference, X-Digital, and Wegener. The format can be = CSI, “CSI”, .CSI, or “.CSI”. (Optional)

MonitorID =

This setting affects the Task Monitor icons that appear at the top of the screen for monitored tasks, and the automatic emails that get sent when a task is not running. (Version 7.0 and above) (Optional)

"MonitorID = Off" - the Task Monitor menu icons are hidden, and the Task Monitor emails will not be sent.

"MonitorID = IconsOff" - the Task Monitor menu icons are hidden, and the Task Monitor emails will be sent when needed.

"MonitorID = EmailOff" - the Task Monitor menu icons are shown, and the Task Monitor emails will not be sent.

“MonitorID = Production” – the Task Monitor emails will indicate they are being generated from the production system.

WebNumber =

An optional parameter used to set the default “web affiliate version number” value for new stations added to the system, whether they’re added manually, or through the Station Information import. The value can be set to 1 or 2. (Version 7.1 and above only.)

ISCIxRefExport =

An optional parameter that can be used to set a custom default export path for the ISCI Cross Reference Export (Version 8.1 and above only.)

[SQLSpec]

Specification for SQL

System =

P7 or P2000. Default is P2000.  Specify which SQL engine is running (P7 for PSQL 7; P2000 for PSQL 2000 or 2000i)



Date =

SQL allowed form of date (P2000 is yyyy-mm-dd;  P7 is either yyyy-mm-dd or mm/dd/yyyy).  Default is yyyy-mm-dd

Time =

SQL allowed form of time (P2000 is hh:mm:ss;  P7 hh:mm:ss).  Default is hh:mm:ss.

[ShowForm]

Specification of data forms for screen display

Date =

Date form on screen.  Default is mm/dd/yyyy.

TimeWSec = 

Time form when seconds are not zero.  Default form is h:mm:ssa/p

TimeWOSec =

Time form when seconds are zero.  Default is h:mma/p.

[Wegener]

Specification for the Wegener Compel Auto-Import

Days_to_Retain =

The number of days to save the import file and SpotOnly.csv file.

Import =

Where the import files are placed for importing.

Save = 

Where the import files are moved after importing.

StationInfo = 

Where the Jones files are stored (JNS_RecGroup.csv, rx_calls.csv, etc.)

[WebServer]

Specifications needed to support the affiliate website (Optional – this section is only added if the Counterpoint Affidavit system is in use)

FTPIsOn =

0 = Off, 1 = On  Use 1 for the web site.

FTPAddress =

Web Server Address 

FTPPort =

Open port number to send and receive via FTP

FTPImportDir =

FTP folder that the web uses to pick-up files from

FTPExportDir =

FTP folder that the Affiliate uses to pick-up web files from 

WebExports = 

Folder that the Affiliate places files being sent to the web

WebImports

Folder that the Affiliate picks-up files from the web

RootURL =

The base path of the web site

FTPUID =

FTP user ID 

FTPPWD =

FTP password

RegSection = 

RegSection is the internal name of your web server as defined during setup. This needed if using the web. 

AllowWebExports =

Setting this to N, No, or NO will prevent exporting to the web. This is only to be used when there is an issue that requires fixing before more web exports can be done. (Optional)

WebServerIPAddress =

Enter the IP address of the web server when using sockets to communicate with the web server. (Version 7.1 and above only)

WebServerPort =

Enter the web server port number (example: 5308) (Version 7.1 and above only)

UsingSockets =

Set to Y (or y) when using sockets to communicate with the web server, or N (or n). (Version 7.1 and above only)

UsingNovelty = 

If set to 1, then web version 3 is used for the CSI Electronic Affidavit system, and several screen changes are applied to the Affiliate system, including handling the website settings and makegood policy settings on the Network Admin page, and not using a single password per station (Version 8.1 and above only)

Test System Information – Affiliat.ini

This section describes the fields on the Affiliat.ini that are used with a test system when the test system is configured to use the special “test locations” section of the Traffic and Affiliat.ini. This is available with Version 7.1 and above only. When configuring a test system like this, the test system path must include the word "test" in it somewhere. For example, d:\csi\test\. This is what tells the system to get the paths from the "Test Locations" section of the Traffic and Affiliat.ini, and to not use the regular "Locations" section. The start in of the test Affiliate system shortcut must be pointing to the location of the test affiliat.ini, which also must be in a folder that has the word "test" somewhere in the path. For example, place the affiliat.ini in the test data folder and set the Affiliate shortcut start in path to: d\csi\test\data. The test locations section of the Affiliat.ini must be filled out correctly. When configured correctly, when launching the test system, the wallpaper will show the word “test system” to make it clear that it’s the test system that’s being used. 


Example test Affiliate system shortcut:


Target: d:\csi\test\exe\affiliat2.exe demo

Start in: d:\csi\test\data


Important note: the shortcut target should include the word “demo” as shown in the example above, as this prevents the Counterpoint Affidavit export from communicating with the web server in order to prevent accidental exports and imports to and from the affiliate website. If a separate test website is set up, and the TestWebServer section is used, do not include “demo” in the target, as the test website allows test web exports and imports to be performed, and for the Unpost and/or Delete Spot utility and the Re-import Affiliate Spot utility to be run on the test website.


On the test Affiliate system, some parts of the system are disabled. The web export does not export to the web (when using Demo mode and when the TestWebServer is not set up), and X-Digital does not connect to the head end - only the test export can be run. 


Note: there is more than one way to have set up a test system. If your test system is set up like a production system, where it doesn't use the TestLocations feature of the Traffic.ini and is simply a separate system set up as a test system, it is not necessary to configure the test affiliat.ini as described here.

Export =

Path used to store test affiliate export files (use CSI\TEST\EXPORT)

Import =

Path for test affiliate import files (use CSI\TEST\IMPORT).

Logo =

Path containing logo files

ContractPDF =

This points to the folder in which PDFs of the contracts between the affiliates and the network are stored.

Timeout =

XXX is the number of seconds to wait without getting a response back from the database to indicate that the database is not responding. If not defined, then the standard timeout is used for Remote Data Objects (RDO) interface to ODBC. If one needs to be entered, start with 5 minutes (300 seconds). The affiliate error message has ‘S1T00 timeout expired’ in message.

MonitorID =

This setting affects the Task Monitor icons that appear at the top of the screen for monitored tasks, and the automatic emails that get sent when a task is not running. (Version 7.0 and above) (Optional)

"MonitorID = Off" - the Task Monitor menu icons are hidden, and the Task Monitor emails will not be sent.

"MonitorID = IconsOff" - the Task Monitor menu icons are hidden, and the Task Monitor emails will be sent when needed.

"MonitorID = EmailOff" - the Task Monitor menu icons are shown, and the Task Monitor emails will not be sent.

“MonitorID = Production” – the Task Monitor emails will indicate they are being generated from the production system.

The recommended setting for the test system is EmailOff

WebNumber = 

An optional parameter used to set the default “web affiliate version number” value for new stations added to the system, whether they’re added manually, or through the Station Information import. The value can be set to 1 or 2. (Version 7.1 and above only.)

[TestWebServer]

Specifications needed to support the test affiliate website (Optional – this section is only added if the Counterpoint Affidavit system is in use and there’s a separate test website set up for testing)

FTPIsOn =

0 = Off, 1 = On  Use 1 for the web site.

FTPAddress =

Test Web Server Address 

FTPPort =

Open port number to send and receive via FTP

FTPImportDir =

FTP folder that the web uses to pick-up files from

FTPExportDir =

FTP folder that the Affiliate uses to pick-up web files from 

WebExports = 

Folder that the test Affiliate places files being sent to the web

WebImports

Folder that the test Affiliate picks-up files from the web

RootURL =

The base path of the test web site

FTPUID =

FTP user ID 

FTPPWD =

FTP password

RegSection = 

RegSection is the internal name of your test web server as defined during setup. This needed if using the web. 

AllowWebExports =

Setting this to N, No, or NO will prevent exporting to the web. This is only to be used when there is an issue that requires fixing before more web exports can be done. (Optional)

WebServerIPAddress =

Enter the IP address of the test web server when using sockets to communicate with the web server. (Version 7.1 and above only)

Enter the test web server port number (example: 5308) (Version 7.1 and above only)

UsingSockets =

Set to Y (or y) when using sockets to communicate with the test web server, or N (or n). (Version 7.1 and above only)

Can be set to Y, Yes, yes, or YES. (If test web vendors will not be used, this line can simply be excluded from the TestWebServer section.) Additional configuration must be done by Counterpoint Software to allow testing web vendors from the test website. In addition, test values provided by the different vendor services must be entered in the Web Vendor Setup screen for each service that will be tested. Note: the AllowWebVendors parameter is only applicable to the TestWebServer section of the affiliat.ini, not the production system “WebServer” section.

Affiliate Test System - Additional Information for Earlier Versions

On version 7.0 and earlier, while there is a separate Test Locations section on the Traffic.ini for a test system, there is no such section available on the Affiliat.ini for those earlier versions. If an affiliate test system is set up, create a test Affiliat.ini with the “Name” pointing to the test database name, and the Locations section pointing to the paths to the test exe, reports, etc. Place the test system Affiliat.ini in the folder where the test Traffic.ini is located (typically the test data folder). When configured correctly, when launching the test system, the wallpaper will show the word “test system” to make it clear that it’s the test system that’s being used.
