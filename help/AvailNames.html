Avail Names

You can create specific avail names to identify, sell, and schedule your inventory. Avail Names will appear on the Spots Screen, and special ‘Book Into’ or ‘Exclude’ dayparts can be defined in Rate Card which will only allow spots to schedule into certain avails.


To enter a new Avail Name, select New from the dropdown, tab, and fill in the following fields:

  • Name: This will be used during scheduling and will appear on the Spots screen
  • Allow Non-Avail Buys
    • If No, only orders selling this Named Avail can book into this avail
    • If Yes, spots sold to a daypart that specifies All Avails can book into this avail (but they will be pre-empted later if someone buys this specific Avail Name)
  • Sponsorship: This is no longer used. Its functionality is now handled through ‘Book Into’ or ‘Exclude’ dayparts
  • Sort Code: Used to sort avail Names on the Avails Combo Report
  • Select on Game Avails
    • If Yes, this Avail Name will appear pre-selected when running the Event Avails report (but can be de-selected)
    • If No, this Avail Name will not appear pre-selected when running the Event Avails report
  • Affiliate System: This controls whether spots with this avail name will be sent to the Affiliate System 
  • ISCI Export: Controls whether spots with this Avail Name will be included on the ISCI Export
  • Audio Delivery: Controls whether spots with this avail name will be sent over audio delivery systems, including X-Digital, Wegener, and OLA
  • Automation Export: This controls whether this Avail Name will be included in the Automation Export
    • If set to No, the spots will be placed in the missed file during the export
    • Setting this to No will also prevent avails from going out over the Engineering Export
  • Filled Break Required: This decides whether an avail must be filled prior to log generation
    • Always: Will require the avails to be filled prior to generating the log
    • Avails default to Always
    • No: Does not require the avails to be filled
    • If Partial: If an avail is partially filled, this will force the remaining seconds to be filled. If the avail is wide open, you will not have to fill it to generate logs
    • If Adjacent Partial: This comes into play if an avail in an adjacent break is partially filled
  • Event Avails Group: available on version 8.1 and above only. This is a toggle field that can be toggled between the hard-coded values of Standard, Billboard, Drop-in, and Extra. These groups are used on the special export version of the Event and Sports Avails report to group spots from different avails together for avails reporting purposes.
