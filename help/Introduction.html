Introduction

This is the help system of the Counterpoint Traffic and Accounting System, Affiliate System, and the Counterpoint Affidavit System. For a brief description of the main screens, exports, imports, and utilities available, see the descriptions below. To access the full help documentation, use the Table of Contents on the left to navigate to different sections of the help documentation.


Traffic Jobs


  • Budgets: the Budgets screen is used to create and manipulate budgets and actuals so that reports can be run to compare what was sold against the plan, and to help determine what the remaining inventory must be sold at to make plan. 
  • Collections: where all accounts receivable activity takes place. Zero Purge and Reconcile are also run from this screen.
  • Copy: the Copy screen is used to manage copy inventory and to create copy rotations. Copy regions can also be defined on the Copy screen, for use with split copy and blackout copy.
  • Invoices: the Invoice screen is used to create preliminary, final, and reprint invoices. Invoices can also be undone on this screen. The backlog screen is also accessed from the Invoices screen.
  • Logs: Traffic logs are created from the Logs screen. The process of generating logs also sends spots to the Affiliate system so that they can be exported to the website using the Counterpoint Affidavit export.
  • Orders: used to enter, modify, and view holds and orders. Orders can also be scheduled from the Orders screen.
  • Post Log: used to reconcile spots after log generation when the spots are in the past. This screen is also used to access the Manual Posting screen for Ad Server contract lines.
  • Programming: used for entering and scheduling program libraries that contain inventory for booking spots and any program elements that must appear on the log. For sports vehicles, the seasons and event schedule are also created here. The Links screen is also accessed from the Programming screen for creating and edited Selling to Airing Links, Delivery Links, and Engineering Links.
  • Proposals: used by salespeople to enter and edit proposals.
  • Rate Cards: the Rate Card Screen is used to define the dayparts that will be sold for each vehicle or standard package, along with their prices if using the Proposal System. 
  • Spots: the primary purpose of the Spots screen is to view and manipulate spots in preparation for closing a vehicle and week. Fill spots can also be created on the Spots screen to fill unsold avails. The Spots screen can only be used to manipulate spots in the future. For spots that have already aired, Post Log is used.

Traffic Exports


  • Accounting: gathers and sends invoices, current and historic collections transactions, and receivables, to a CSV file that can be imported into a general ledger system.
  • Ad Server Billing Discrepancy: The Ad Server Billing Discrepancy Export is run by month and year and is used to compare the number of impressions ordered against the number of posted impressions, calculate the difference, and show a status of “Missing” if there are no posted impressions for the month, for ad server lines.
  • AirWave: produces two comma-delimited files, one of which contains spots, the other contains controls, for the final network broadcast schedule.
  • Audio ISCI Title: exports a list of copy for the selected vehicles using rotation entered dates, and/or log dates.
  • Audio MP2: exports a list of all copy running between a specified date span on selected vehicles, which is used to compare the copy names in Counterpoint to those in X-Digital, to ensure that the names are identical.
  • Automation: there are numerous automation exports available, which compile and store spot information in a file that is imported into automation systems, such as Dalet, Drake, Enco, and so on. For a complete list of automation exports, along with the specifications of each export, please see the Traffic Exports help document that is available on the Counterpoint website.
  • Barter Payment: sends rep barter payment information to the Great Plains system.
  • Carts: lists all active, purged, and/or historic copy with a requested date span for an individual or group of vehicles, and is intended to be used in conjunction with the X-Digital Cumulus export, when dual head ends are used.
  • Cash Receipts: includes cash information consisting of Payment (PI), Payment on Account (PO), and Journal Entry (W) transactions, which will be gathered using the transaction date against the dates entered. 
  • Clearance and Compensation: used to send advertiser and product information, inventory/avails, selling to airing links, and spot information to an independent system.
  • Commercial Changes: tracks spot moves and lists the original “Moved From” date/time/vehicle and the last “Moved To” date/time/vehicle.
  • Contract Line: gathers all air time and NTR lines from contracts that are active within a selected date period and exports the data to a CSV file. Similar to the Paperwork Summary report.
  • Copy Bulk Feed: sends cart information to stations based on a code entered in the Vehicle Options screen, Bulk Feed Group Number field.
  • Corporate: converts revenue and inventory information into a fixed format that is imported into a corporate database.
  • Custom Revenue: a revenue export similar to Matrix that is used to export revenue into an external system.
  • Dallas Feed: a client specific export used to create the Dallas Studio Log.
  • Efficio: there are two Efficio CRM exports: Revenue and Projections. The revenue and projections export can be generated by standard broadcast or calendar months. For standard broadcast months, the revenue export gathers receivables information from billed months, and the projection export gathers contract information from unbilled months, starting from the month after the last billed and closed month up through the last month in the future with contracts ordered for it. For Calendar month exports, both revenue and projection data comes from schedule Order lines. 
  • Enco: creates one export record for each day, time zone (time zones marked with an asterisk) and vehicle, along with a PDF log for verification of the export files. All time zones are combined in this export that contains the feed time (adjusted by the vehicle zone table), advertiser/product, spot length, ISCI, creative title and time zone.
  • Engineering Feed: feeds spot information into a proprietary engineering system.
  • Export Inventory and Spots: a standalone export program that creates a Spot Export file and an Inventory Export file that can be imported into a Hyperion database.
  • Get Paid: Get Paid is a third party application used for collections purposes. The Get Paid export creates two files: ARMAST.txt, which contains one record per customer and network specific information about that customer, and ARMCUST.txt, which contains one record per transaction (invoices, journal entries, and payments).
  • Great Plains: creates two accounts receivables files to import into a Great Plains database.
  • Invoice Export: extracts invoice and spot data, for “as aired” billing only. When this feature is used, the Invoice Project Code and Invoice SMS# fields are shown in the contract number popup box.
  • Invoice Register: includes billing information consisting of Invoice (IN) and Invoice Adjustments (AN) transactions, which will be gathered using the transaction date against the dates entered. 
  • Matrix: a revenue export imported into the Matrix Media CRM and Sales Analytics Software.
  • Miller Kaplan X-Ray: generates up to 24 months of billing, by agency, advertiser, and vehicle, in CSV format.
  • Phoenix Log: a network specific export that is used to send spot information to a proprietary system. Vehicles included in this export must be using Delivery Links.
  • Proposal XML: exports Counterpoint proposals to agency systems in XML format.
  • RAB: the RAB export is a revenue export that is compatible with the Radio Advertising Bureau CRM system. It has three ways it can be run: by calendar month, by standard broadcast month, or by calendar spots. 
  • Revenue: creates a CSV file of the data that is generated by the Billed and Booked Report. The file contents can be imported and sorted various ways, for reporting needs beyond the Billed and Booked options.
  • Sales Force: a standalone program that exports past revenue and future spot and NTR projected revenue for importing into the Sales Force CRM application.
  • Station XML: an export that is similar to the Proposal XML export but that is used with Wide Orbit, and that creates a separate XML file for each vehicle.
  • Tableau: a revenue export that gets imported into the Tableau Media CRM and Sales Analytics software program.
  • vCreative: the vCreative export automatically sends new and modified copy to vCreative when exiting from the Copy screen after making certain copy changes, such as creating a new rotation or changing the dates of an existing rotation.
  • WO Invoice Export: generates an invoice export in XLS format for importing invoices into an external system. 

Traffic Imports


  • Automation imports: spot airing information can be directly imported from automation systems for the following automation systems: Audio Vault Sat, Jelli, Prophet NextGen, and others. 
  • Filemaker: used to import new orders from Filemaker. When enabled, new orders can only be imported from Filemaker, they cannot be entered manually, although the Traffic system is used to revise existing orders.
  • Research: there are five methods of importing Research data into the Traffic system. Act 1 is used to import data from Act 1. The CSV, RADAR, and Vehicle (Edison) imports are simply other formats that can be used. The Impressions research import is only used with systems with the Traffic Site Options setting of “Show ‘Impressions’ and ‘Download’ on Proposals/Orders” checked on. For this import, the imported data is stored with the demographic category of P12+ (no other demographic categories are used with this import), and a special screen appears on the Research screen for books that have been imported using the Impressions research import.

Traffic Utilities


  • Archive: gathers data prior to a defined archive date, removes it from the database, and either stores it in external files where it can be retrieved if needed, or purges the data.
  • Archive Retrieval Utility: extracts archived data from archive files for a single advertiser within a requested date span and places it into a CSV file.
  • CFF Check: verifies schedule lines and weekly flights for consistency.
  • Compare Act 1 Lineup to Packages: compares the stations that make up a standard package defined in the Traffic system to the stations detected on an Act 1 Lineup PDF file. This is only enabled if stations are defined as vehicles, so this utility is only needed and available under special circumstances.
  • Contract Scheduler: a program that runs in the background and that automatically schedules contracts that require scheduling.
  • Contract Swap Check: checks and fixes contract swap errors, where contract lines or flights are not pointing to the correct internal contract revision number.
  • Contract Verification: this is a screen that allows designated users to track and alter a contract’s status during the agency verification process. The three statuses are Not Verified, Sent to Agency, and Verified.
  • CSI Backup: a backup program used to back up the Traffic and Affiliate database.
  • CSI Copy Database: copies database backup zip files from a source folder to a destination folder. It both copies database backup files and verifies that the file was copied successfully and is valid.
  • CSI Server Manager: used to manage the CSI_Server program that can be used to run scheduled tasks.
  • Days to Pay Check: checks and fixes three Advertiser and Agency values: average number of days to pay; number of days to pay last payment; and the number of invoices paid.
  • DB Unzip: unzips backup databases into the correct location.
  • DDF Reorganize: compares and updates DDF files to ensure all files, dates, and times are in sync with one another. The DDF Offset utility is also used as part of this process.
  • Export SQL: extracts spot, daypart, advertiser, regional copy, and vehicle data from the database and either sends it to an SQL server or to a CSV file.
  • File Check: a standalone utility that checks system files and repairs issues.
  • Find: the Find button is a global way of locating scheduled spots for a span of dates for a specific contract.
  • Generate ASF: a standalone utility that can be run periodically and that is used to generate six months of avails information for the “Six Months Avails” report.
  • Invoice Check: compares contract dollars with receivables dollars updated by final invoicing, for cash portions of contracts only, excluding NTR, installment billing, and rep contracts.
  • Messages Viewer: used to view log files that are created when doing tasks such as running an export, running logs, etc.
  • Reset Work Files: some Counterpoint reports and processes use a work file to store data gathered from multiple files before writing it to the display, printing it, or saving it as a file. These work files are automatically reset when the Counterpoint Backup is run, but they can also be reset manually using this task.
  • Set Copy Inventory: checks the earliest start date and the latest end date for active copy rotations, and adjusts any date discrepancies that may exist.
  • Set Credit: updates the credit limit values as shown on the Advertiser and Agency Credit Status report.
  • SMF Check: checks the integrity of the scheduled spots against the missed spots.
  • Sports Contract Check: verifies that the game schedule defined in programming and the game dates retained with the contract match.  
  • SSF Check: compares the spot summary file (SSF) with the spot detail file (SDF), and looks for discrepancies. When it finds them, it will log the vehicle, date, and type of error found.
  • Station Spot Builder: a standalone utility that creates Affiliate Spots (AST records) along with the associated copy after the log has been generated in the Traffic System.
  • System Shutdown: can be run from the Accessories menu to initiate a system shutdown so that software can be updated or to perform other maintenance.
  • Task Monitor: runs in the background and checks the status of various tasks that either run periodically on a schedule or run continuously, and shows the status of each monitored task in the title bar of the Traffic and Affiliate system. Tasks that require monitoring are configured using the Monitor Setup program.
  • Traffic Alerts: Counterpoint has a built in alert system that notifies designated users of tasks that need to be performed, such as reviewing a completed proposal, scheduling contracts, or reprinting logs after changes were made. If set to receive alerts, the user will see a red flashing Alert button at the top of the Counterpoint screen. Clicking on the Alert button will display the action that triggered the alert, from which actions can be taken to resolve the alert.
  • User Status Screen: this shows who is signed on to the Traffic and Affiliate system, when a user was last on the system, how long they were on the system for, and is also used for initiating system shutdowns.
  • View Blocks: system blocks (or locks) are used to prevent multiple users from altering the same record at the same time. The View Blocks tool allows you to see and remove blocks.
  • View SQL: allows users to execute SQL statements on the Pervasive database to review or correct records.

Affiliate Jobs


  • Affiliate Affidavits: the Affiliate Affidavits screen allows you to view affiliate logs and post their affidavits in the case of manually posted agreements. This screen can also be used as a troubleshooting tool, for viewing affiliate spot information at the spot detail level.
  • Affiliate Agreements: the contracts between the network and affiliates are defined and edited here. The agreement will state what the affiliate will carry, how they will receive their spot information, along with the dates of the agreement, and other information.
  • Affiliate Management: allows affiliate users to view all information about stations and agreements, including aired compliance, comments and action items, all on one screen.
  • Emails by Vehicle: mass and individual emails can be sent to affiliates directly through this screen. Welcome, Welcome by Vehicle, Password, Overdue, and Custom emails are available.
  • Export Center: allows users to define, manage, and queue exports for running in the background based on priority numbers assigned to the different exports.
  • Network Log: the Network Log screen is used to view the spots and copy on the Affiliate system based on what was generated during the Traffic log generation process. Changes can also be made to the spots prior to exporting to log or audio delivery services.
  • Post Buy Planning: this screen is used when performing a post buy analysis to gather data about where spots for a specific advertiser aired for a specific week.
  • RADAR: used to configure the RADAR export.
  • Site Options: Affiliate Site Options such as general options, Emails by Vehicle screen email text, and web options, are configured here.
  • Stations: used for viewing and editing station information.
  • User Options: Affiliate system users and their permissions are configured on this screen.


Affiliate Exports


  • Affidavit Overdue: produces a count of overdue affidavit weeks for the selected vehicles using the entered date range. Each record includes the Station ID, the Call Letters-Band, the number of overdue affidavit weeks and the vehicle name. 
  • Clearance and Compensation: This is a network specific export containing aired spot information that is imported into a proprietary system. 
  • Counterpoint Affidavit Export: used to export affiliate spot data to the Counterpoint Affidavit website.
  • IDC: produces a file that tells the IDC system (International Datacasting Corp.) what split copy to air.
  • ISCI Export: sends commercial audio providers a list of the ISCIs to air. There are two methods available: by Unique ISCI (Mr. Master), or by Break.
  • ISCI Cross Reference: used with the X-Digital system to ensure that engineering and continuity are using the same names when labeling client commercials. 
  • Jelli: an export that is created during the Counterpoint Affidavit export process when the Jelli export feature is turned on. The contents of the export includes copy information, dates, spot length, advertiser name, etc.
  • Label Info: produces a CSV file that can be opened in Excel and used to generate mailing labels. The information is pulled from the Stations Personnel screen.
  • Marketron: sends Affiliate spot information from the network to Marketron stations.
  • RADAR: generates a file that is sent to RADAR that is used for advertiser clearance information. The file contains the program schedule information, plus the actual air time and day of the spots within a specified week.
  • RCS 4 and RCS 5: sends spot and copy information to RCS for the selected date, time zones, and vehicles.
  • Starguide: can send all spots or regional spots only for use with importing into the Starguide system.
  • Station Compensation: creates an export file that can be imported into a compensation system to determine the amount due to a station and/or network for three different compensation methods: barter, affiliate, and network.
  • Station Information: creates a CSV file containing station information, such as call letters, market, state, etc.
  • Univision Scheduled Station Spots: allows network spots to be exported from Counterpoint and imported into the Univision Stations Marketron Traffic system. Aired information can then be imported back into Counterpoint.
  • Wegener Compel: the Wegener Compel Network Control system is an Audio delivery system that allows a network operator to control station receivers and uplinks. The Counterpoint Affiliate System produces an ISCI file containing ISCI and insertion codes that is used by the Compel system to send generic and regional copy to stations.
  • Wegener iPump: Wegener iPump is an audio delivery system that controls station receivers and uplinks. The Affiliate system produces a playlist file for each station.
  • X-Digital: X-Digital Systems, Inc. is a system that delivers commercial and programming audio to affiliates to enable regionalized and localized ad campaigns. Specific advertising spots can be sent to regions or specific receivers. Audio files stored in the receiver can be inserted into live or recorded programs. With the X-Digital export, the Counterpoint system can send the following: spot insertions, file delivery, station information, agreements (authorizations), and vehicle (program) information.

Affiliate Imports


  • Affiliate Pledge Import: imports pledge information for agreements that have been previously added to the Affiliate system but with the pledges not previously defined.
  • Agreement Data: a network specific import that pulls pledge information into the Affiliate system from other data sources. Imports Air Dates, Pledge Times, and CPs.
  • Auto Import: this task can be run on a schedule to automatically import completely posted affidavits from the Counterpoint Affidavit system.
  • Counterpoint Remote Traffic Spots: transfers spots from one Counterpoint Traffic database to another Counterpoint Affiliate database.
  • Counterpoint Affidavit Import: imports completely posted affidavits from the Counterpoint Affidavit system.
  • Marketron Import: imports affiliate airing information directly from Marketron.
  • Spot Data: updates the Affiliate system with spot and agreement data from other data sources. There are three types available: Log, Affiliate, and MYL.
  • Station Import: imports new or updates existing station information in the Affiliate database. Data can be imported from Act 1, BIA, or using a custom CSV file.
  • Univision Aired Station Spots: imports aired spots from stations to the Affiliate system.
  • Wegener Compel Import: this imports airing information from Wegener Compel, and also updates the dates and times of web spots when those dates and times have been captured by Wegener Compel. Auto-posted spots are shown on the website, and station users can manually post spots that were not updated by the auto-posting feature.
  • X-Digital Import: imports airing dates and times and copy information directly from X-Digital, and posts the airing information directly on the Counterpoint Affidavit website, so that station users do not have to manually post the airing information for those spots.

Affiliate Utilities


  • Affiliate Export Queue: allows Affiliate exports to be queued up and run in the background.
  • Affiliate Measurement Utility: this a standalone background program that runs at a designated time at least once a week. The utility amasses compliance information that is stored in the Affiliate Measurement file. The file contains 52 weeks of information for every agreement in the system. It’s used with the Affiliate Measurement report, which shows the data in report form.
  • Affiliate Report Queue: allows Affiliate reports to be queued up to run in the background, so users can accomplish other tasks while reports are generating.
  • Alerts: alerts are used to notify affiliate users that certain actions have happened, and that they should take certain actions in response to the alert. The following affiliate alerts are available: Export Spots, Export ISCI, Agreement alerts, and Web Vendor alerts.
  • AST Check Utility: checks if AST records are in sync with Affiliate Agreements, and checks the integrity of affiliate posts.
  • Calculate Average Weeks Delinquent in Posting: shows the average number of affidavit weeks that are unposted or partially posted per agreement.
  • Check Email Format: examines all of the email addresses entered in the system, and verifies that they are all formatted correctly. It does not check that they are currently being used, but rather that they meet the correct format for email addresses.
  • Clean-up Station Info: scans the system for any duplicate Formats, Markets, Owners, Territories, and for any unused Markets and/or Formats, and removes any that it finds.
  • Compliance Tracer: analyzes how the Affiliate Station spots are obtained and how the spot counts have been determined.
  • CPTT Agree Between Affiliate and Web: this has not been implemented.
  • CPTT Check: adds missing CPTT records and removes extra CPTT records. 
  • CPTT Rectify: creates missing CPTT records as well as missing Affiliate spot records (AST records).
  • CSI Marketron Helper: Marketron allows no more than 1,000 files to be imported in a single session. The CSI Marketron Helper Utility will assist those with large numbers of import files that encounter this limitation.
  • Duplicate Blackout Fix: fixes duplicate blackout spots that were being created under specific circumstances (when an affiliate spot report was run for the generic advertiser only, for a vehicle/station/week with blackouts, where the affiliate spots had already been created).
  • Duplicate CPTT Fix: searches the entire database for any duplicate CPTT records for a single affiliate and agreement. If any are found, the utility automatically fixes them.
  • Messages Viewer: used to view log files that are created when doing tasks such as running an export, running logs, etc.
  • Re-Import Affiliate Spots: used to re-import posting information directly from the Counterpoint Affidavit system and from Marketron Direct Connect. This utility can only be run to re-import spot information that has previously been imported; it will not import data that has not yet been imported. The purpose of re-importing a previously imported week would be to correct an error with the posting information that occurred after importing the spots from either the Counterpoint Affidavit website or Marketron Direct Connect so that the airing information reverts back to what was originally posted by the affiliate.
  • Set Compliance: can be run to set the compliance flags for affiliate spots that were created prior to converting to Version 7.0. (Spots created in Version 7.0 and above will automatically have their compliance flags set when posted spots are imported.)
  • Set CPTT Complete: a standalone application that verifies the posting status of each affiliate agreement. When it is generated, all posting is checked, and the posting status is assigned one of the following statuses: Completely Posted, Partially Posted, or Not Posted. If a CPTT record is set to “Did Not Air,” but there are spots associated that did air, the CPTT record will be marked as “Partially Posted” or “Completely Posted,” depending on whether its spots are partially or completely posted.
  • Set MGs: the Set MGs (set makegoods) utility is used to set a “makegood” flag on affiliate spots that were marked by a station website user as airing outside of the pledge week and were imported in the past (prior to this feature being implemented). It also creates a missed spot for the original fed spot so that each makegood has a missed spot that it ties back to. 
  • Set Program Times: sets the program time range (start/end times) for each affiliate agreement. These times are then used to help in determining the Station Compliance for each affiliate. 
  • Spot Count Tie-Out: The Spot Count Tie-Out utility is used to examine the current spot counts for a single week for troubleshooting purposes. Spot counts by network, feed, pledge, affiliate spot, web makegoods, vendor exports and imports, and network and station compliance counts, are all shown and can be analyzed.
  • Unpost and/or Delete Spot: there are four different ways of running this utility: “Clear Posted Spots” clears an erroneous posting done by an affiliate; “Delete Posted Spots” removes affiliate system spots, web spots, and resets the Affiliate Affidavit status to outstanding; “Delete All Spots for All Weeks” removes all spot information from the Counterpoint Affidavit system and from Affiliate Affidavits, as if the spots were never generated; “Delete Network Web Spots” removes all spots from the Counterpoint Affidavit system, but not from Affiliate Affidavits.
  • User Status Screen: this shows who is signed on to the Traffic and Affiliate system, when a user was last on the system, how long they were on the system for, and is also used for initiating system shutdowns.
  • Vendor Setup: used to indicate whether various log delivery services and audio delivery services are active or dormant, and for storing connection information for active services when needed, as well as for configuring delivery vendor options. 
  • View Blocks: system blocks (or locks) are used to prevent multiple users from altering the same record at the same time. The View Blocks tool allows you to see and remove blocks.
  • View SQL: allows users to execute SQL statements on the Pervasive database to review or correct records.
