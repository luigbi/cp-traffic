Get Paid

Get Paid is a third party application that is used for collection purposes. There are two ways to run this feature. You can select to export the data on demand by choosing Task ->Export -> GetPaid, or schedule the export to run at a predetermined date/time using CSI Server Manager. 


The GetPaid export creates two files and places them in the csi\prod\export folder. The files are named ARMAST.TXT and ARCUST.TXT. 

  • The ARCUST.TXT file contains one record per customer and specific information about that customer
  • The ARMAST.TXT file contains one record per transaction (invoices, journal entries, and payments)

Setup

  1. Activate “Get Paid Export” in Site Options -> Options.
  2. Enable the Service Manger on the server.
  3. Add Get Paid to the CSI_Server.ini, located in the csi\prod\exe directory, to run the Get Paid export automatically.
  4. Set up the terms for the ARMAST.TXT file, such as “Due 15 days upon receipt,” in the Terms field in the Agency or Advertiser (Direct Advertiser) screens.
    1. The terms will appear in columns 613-647.
    2. If there is a numeric value in the terms, it will appear in columns 69-72.
      1. Note that the number of digits in the terms should not exceed 3. For example, 999 would be okay, but 1000 would exceed the limit.
    3. If there is not a numeric value, a “0” will appear in columns 69-72.
    4. If there are no terms defined on the advertiser/agency level, the terms shown will be pulled from the Site Options -> Export tab -> Default Terms field.

Example CSI_Server.INI

/IniLoc: X:\CSI\Prod\Data

LogFile = X:\CSI\Prod\Data\Messages\CSI_Server_Log.txt\

[GetPaid]

Enabled = 1 

Weekdays = 0111111

StartTime = 05:00am

Interval = 86400

EXE = X:\CSI\Prod\EXE\GetPaid.EXE

StartInFolder = X:\CSI\Prod\Data

CommandLine = /INILoc:X:\CSI\Prod\Data

KeepProgramRunning = 0
