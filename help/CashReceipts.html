Cash Receipts

This export includes cash information consisting of Payment (PI), Payment On Account (PO), and Journal Entry (W) transactions, which is gathered using the transaction date against the dates entered. These include Air Time & NTR, Cash, Trade, Political, and Non-Political types.


This export can be run on demand using any date span from the Task -> Export -> Cash Receipts menu.


Cash Receipts filename example: 

Cash Receipts mmddyy-mmddyy cccccc.csv 


The words “Cash Receipts” is hard coded, mmddyy is the start date entered, mmddyy is the end date entered, and cccccc is the Client Abbreviation Name. Exports generated multiple times on the same day will append a letter, starting with “A”, to the filename.


File Format Specifications

  1. Information is stored into comma-separated variable files.
  2. All alpha fields are in double quotes.
  3. Dollar amounts contain decimal points only if pennies exist. If dollars exceed 999.99, commas are omitted.
  4. The data is unsorted.
  5. One line per transaction is created.
  6. Line 1 of the export contains the field heading names.
  7. Balancing totals appear on the last line(s).
