Network Admin and Settings

This section explains the configuration options for the Counterpoint Affidavit System Website Version 3. 


The Counterpoint Affidavit System settings that were previously configured on the Affiliate Site Options screen, including Web Options, Web Makegood Policy rules, the Citation text, and the Network Administrator contact information, have been removed from the Affiliate Site Options screen and moved to the new Network Admin pages as part of the suite of improvements made for version 3. A Network Admin account is required for accessing the Network Admin pages. 

Network Admin

When signing onto the Counterpoint Affidavit System using a Network Admin account, a Settings button will be available at the top of the screen, which is used to for accessing the Settings page. It can also be accessed by clicking your profile in the upper right corner and selecting "Settings". 


Network Admin accounts can also view the logs and affidavits for every affiliated station by selecting the station from the station dropdown list that appears on the Dashboard page. This station dropdown only appears for Network Admin accounts. It does not appear for station user accounts (unless the station user account is associated with multiple stations. When this is the case, the station dropdown list will appear and it will show that user's associated stations).

Settings

There are five tabs on the Network Admin Settings page:


  • User Accounts: create, modify, and deactivate Network Admin user accounts
  • System Settings: configure system settings such as whether makegoods are allowed 
  • Makegood Policy: configure web makegood policy rules
  • Citation: configure the custom Citation text that appears when a station user begins the process of posting an affidavit
  • Admin Info: configure the contact information for the System Administrator

User Accounts

The User Accounts tab is used to create and modify Network Admin user accounts. Network Admin accounts are network personnel who manage and use the Counterpoint Affidavit system. Network Admin accounts are not for station personnel. Station user accounts are used by station users and grant access to the data for one (or more) stations, while a Network Admin account has access to the data for all affiliated stations.


The following information is available on the User Accounts tab:


  • User name: uses the Network Admin's email address
  • First name
  • Last name
  • Title
  • Phone
  • Status: a Network Admin is either Active or Dormant.
  • Action: the following actions are available:
    • Edit: edit the user account information.
    • Resend password reset link: clicking this button will send the user an email with a link they can click to reset their password. 
    • Delete: delete the user account. 


The user name, first name, and last name, are all mandatory fields.


Note that a logged in user cannot resend themselves a new password link or delete their own account. These buttons will not be present for a logged in user's own account.

To add a new Network Admin user account:


  1. Click the Add User button in the upper right corner.
  2. Enter their first name, last name, and email address.
  3. Optional: enter their job title and phone number.
  4. Press Save. This will trigger an automated email that gets sent to the new user's entered email address. They must then click the link and enter a password to complete the process of creating their account. Note that the link expires in 24 hours. If they have not confirmed their new account within 24 hours, the "Resend password reset link" can be used to send them a new link.

System Settings

The System Settings tab is used to configure settings used by the Counterpoint Affidavit system.


The following settings are available:


  • Number of days to retain posted and exported spots on the web before erasing: The amount of time (in days) you will allow posted affidavit information to remain on the website before it is permanently erased. (This option is deactivated, spots are not erased from the website.)
  • Number of days to delay the network from importing posted web spots (0-30): The amount of days that a posted affidavit will remain on the Counterpoint Affidavit System before it is imported into the Network Affiliate System. Delaying the importing of an affidavit gives the affiliate time to make changes if necessary. This will affect all agreements. The max setting is 30 days.
  • Number of days to retain missed spots on the web: This determines how long a missed spot remains on the web before it gets archived. (This setting is currently deactivated.)
  • Number of days to view posted spots on the web: This determines the number of days that posted spots will remain on the Counterpoint Affidavit system before they are archived (and no longer able to viewed by affiliates). This number must be equal or greater than 14.
  • Suppress asking missed reason: When enabled, spots on the web can be marked as not aired without specifying a missed reason. Note that this option cannot be enabled if "Allow web makegood spots" is enabled. In other words, if web makegoods are allowed, then missed reasons are required.
  • Allow web makegood spots: When enabled, makegoods can be created on the Counterpoint Affidavit system by station users of the website. 
  • Allow posting on today and on future days: When enabled, future spots and spots on today's date (or in the past) can be posted normally. When this option is not enabled, then when viewing affidavits on the station website, any spots pledged to air today or in the future will have their "Aired" status field and "Air time" appear as unavailable, which prevents them from being posted. Once those spots are in the past, those fields will be available and can be posted normally. 
  • Keep multicast in sync: When not enabled, after a web affidavit has been posted for a multicast agreement, changes can be made to the posted spots that do not get copied to the other station or stations in that multicast agreement. When enabled, after a web affidavit has been posted for a multicast agreement, changes made to the posted spots will get copied to the other station or stations in that multicast agreement, keeping the multicast affidavits entirely in sync.
  • Enable color coded changes: When enabled, the feature that displays removed spots on the web log in a red font color, and added spots and changed copy in a green font color, will be used. When not enabled, removed spots will not be shown, and the green font color for added spots and changed copy will not be used. 


When making changes, press the Save button to save the changes. The Reset button will revert changes back to the last saved setting.

Makegood Policy

The settings on this tab determine the rules for web makegood creation. For a makegood to get created successfully, the user-entered makegood information must pass the selected rules. If a station website user attempts to enter a makegood that would violate one or more of these rules, they will see a warning message that explains the first rule violation. For example, if the makegood setting “must air within missed standard broadcast month” is violated by entering a makegood date outside of the standard broadcast month that the spot was missed from, a warning message will appear that states that the makegood must be in the same standard broadcast month as the missed spot. Note that a single spot could violate several rules.


  • Makegood  Policy
    • Must air within Standard Broadcast Month: When this is selected, to create a makegood, it must be given an air date within the same standard broadcast month as the original missed spot. If this option is selected, the setting “Miss in last week of broadcast month, makegood OK in next week” below becomes available. 
      • Miss in last week of broadcast month, makegood OK in next week: If a spot was missed in the last week of the standard broadcast month, when this is checked on, it can be made good in the first week of the next broadcast month. (This option is only available if the “Makegood must air within standard broadcast month” radio button is selected.)
    • Must air within X days of missed date: When this is selected, to create a makegood spot successfully, it must be given an air date within the number of days specified here.
    • Can air on any date: When this is selected, when creating makegoods, the air date can be any date as long as none of the other date rules are violated (such as the “book only into order flight days” rule), and there is at least one spot from the same contract airing in the intended makegood week.
  • Disallow MGs in hiatus weeks: When this is enabled, there must be at least one other spot for the same contract airing in the makegood week. If there isn’t at least one spot, the week is considered a “hiatus” week and the makegood cannot be created for that week.
  • Book only into order flight days: When this is enabled, when creating a web makegood, the makegood air date must be on one of the valid days of the week for the missed spot, using the daypart defined for the contract line.
  • Time Restrictions: The Time Restrictions settings determine what makegood times are accepted. A warning message will appear if a station user attempts to enter a makegood time that violates the time restriction rule.
    • Order Flight Times: When set to Order Flight Times, the makegood aired time must be within the ordered flight times (as defined on the contract daypart for exact time spots, or within the station pledge time range for ROS spots). 
    • Station Pledge Times: the makegood aired time must be within the agreed upon pledge time range for ROS spots, or within a standard daypart for exact time spots. 
    • Day Split: With the "Day Split" setting, spots missed between 12m-5a can be made good at any time. Spots missed between 5a-12m can only be made good between 5a-12m.
    • Any Time: any makegood time is accepted.
  • ISCI Restrictions: The ISCI Restriction setting is used to determine what rules apply to the makegood ISCI code.
    • Same Order: the makegood ISCI must be the ISCI from the contract. 
    • Same Order, then Same Advertiser: if there’s no valid ISCI for the contract, it will be possible to select another ISCI for the same advertiser but from a different contract.
    • Same Order + New: the user can select an ISCI from the same contract, or enter a new ISCI code.
    • Same Order, then Same Advertiser + New: the user can select an ISCI from the same contract, or from the same advertiser but from a different contract, or manually enter a new ISCI code.
  • Competitive Separation Time in Minutes: The number of minutes specified here will appear on the “Makegood Policy” shown on the CSI Electronic Affidavit website when creating makegoods.
  • Allow Bypassed Makegoods: When enabled, stations users will have the ability to create a "bypassed makegood" from a missed spot, which indicates that it will never be made good. When this option is deactivated, the "bypassed makegood" feature will be entirely hidden from station website users.


When making changes, press the Save button to save the changes. The Reset button will revert changes back to the last saved setting.


Station users will see the selected rules on the website Makegood Policy screen, which is viewable from the screen where makegoods are created from.

Citation

The Citation tab is a text entry field that is used for entering the citation text that a station user must agree to before they can post affidavits.


Press Save to save changes, or press Clear to clear the text.

Admin Info

The Admin Info tab is used to configure the contact information that is displayed when someone clicks the "Help me sign on" button on the login page.
