Podcast Rate Card

The Rate Card screen is used for entering the vehicles and dayparts/ad locations for podcast and ad server vehicles, the default CPM for Digital lines, and the default rate card price for podcast spot buys. Podcast/Ad Server vehicles are shown in italics.


When the Ad Server Site Option setting is enabled, the CPM field will be shown on the Rate Card screen, as shown in the picture above. This field is only accessible for vehicles set to the Podcast/Ad Server medium type, and is used to set the default CPM for the Digital lines using this vehicle/daypart (ad location) combination. In the example above, for Podcast A, the default CPM for the Pre-roll ad location is $20.


The default rate card spot price can also be set in the price field for vehicles/dayparts used for podcast spots.

Daypart/Ad Location

In Counterpoint, the Daypart field on the Rate Card is used to specify the Ad Location for Podcast/Ad Server vehicles. For spot buys, the daypart days and times are used when scheduling the spots. The “book into” feature can be used to ensure spots schedule into the matching avail name. For example, you can specify a “book into” avail name of “mid-roll” for the Mid-Roll daypart/Ad Location so that those spots will book into the correct avail.


For Ad Server buys, the Daypart/Ad Location is used to determine the ad location that the line is targeting. A single Daypart can be used for multiple Ad Locations, for example, Pre-roll and Mid-roll on a single Daypart. When setting up Dayparts/Ad Locations for Ad Server vehicles, the valid days and times can be defined as Mon-Sun, 12m-12m.


For additional information on setting up Dayparts, see the Rate Card help document.

CPM Standard Packages

Standard packages that use the CPM price type can be created and edited on the CPM Package screen, which is accessed by pressing the CPM Package button at the bottom of the Rate Card screen.


CPM Standard Packages are used to add a CPM package to a contract that uses a pre-defined set of vehicles and ad locations. When adding a CPM Standard Package to a contract, it distributes the total impression goal entered for the package to the hidden lines that make up the package, using the impression percentages entered on the standard package screen. In the example above, 25% of the total package impressions will go to Podcast A Pre-Roll, 25% will go to Podcast A Mid-Roll, and so on, for the full 100% of the package impressions.


To create a CPM Standard Package:

  1. Select New from the package dropdown in the upper right corner of the screen.
  2. You can press Tab to model from an existing package, or simply start entering the package information to create a new standard package from scratch.
  3. Enter the standard package name in the “CPM Package Name” field.
  4. Enter an “Impression %” for each vehicle and ad location combination (pulled from the Podcast/Ad Server vehicles from the Rate Card) to include in the package. This value must add up to 100% before the package can be saved (a total percentage is shown at the bottom and will turn from red to green when it adds up to 100%).
  5. Press Save when finished.


After the standard package has been created, it must be added to the Rate Card to allow it to be used on contracts. To add a standard package to the Rate Card, simply select it from the vehicle dropdown, enter a daypart, set the Base and Report value to N, enter a sort number if needed, then Tab to the next line and press Save.


Standard Packages on the Rate Card are shown in a blue font.
