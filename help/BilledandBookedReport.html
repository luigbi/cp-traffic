Billed and Booked Report

The Billed and Booked report shows advertising revenue for the selected period for up to one year of data. This section provides an explanation of each selectivity option on the Billed and Booked report.

Selectivity Options

Year: Enter the year the report should be run for. It can be entered as a two or four digit year.


Star Month: Enter the month that the report should start from. Use a one or two digit month or the three letter month abbreviation.


# Months: a maximum of 12 months can be entered.


Sort: Select the sort options from the list of available options.


Totals by: Choose the level of detail for the total values. The three available options are (in descending order of detail): Contract, Advertiser and Summary.


Month: Choose the type of month to use.

  • Corp: Corporate calendar. This option is only available when a corporate month is defined in Traffic Site Options. Revenue information is gathered from the contract, not from receivables when using a corporate calendar. When run to include invoice adjustments, it includes invoice adjustments entered on or prior to the last invoice date.
  • Std: Standard broadcast calendar. For periods that have not yet been billed, the report gathers data from the contract. For billed periods, it gathers data from receivables. When run to include invoice adjustments, it includes invoice adjustments entered on or prior to the last invoice date.
  • Cal: Calendar month. This option is typically not used. When selected, it uses the regular non-broadcast calendar with spot rates averaged across the week (because it does not take spot moves into account and it’s possible for spots from a single line to go across the calendar month). With this selection, only contracts are used, not receivables. When run to include invoice adjustments, it includes invoice adjustments entered on or prior to the last day of the calendar month that final invoices have been run for.
  • Cal (Spots): Calendar Spots version. Use the regular non-broadcast calendar with spot moves reflected. This version can be used to compare to the Matrix-Calendar export. Note: if balancing to the Matrix-Calendar export, the Export Net/T-Net option, Start Month, Start Year, # of Months, NTR and Missed Spot settings must be set to the same values in order to balance like to like. When run to include invoice adjustments, it includes invoice adjustments entered on or prior to the last day of the calendar month that final invoices have been run for. 
  • Bill method: On version 8.1 and above, if both the standard broadcast calendar billing cycle and the calendar billing cycle are used, the Billed and Booked report can be run by “Bill Method” to gather data by both billing cycle methods, based on the billing cycle used for each relevant contract. For contracts on the standard broadcast billing cycle, the Bill Method will work like the standard broadcast calendar method, pulling data from contracts for unbilled periods, and from receivables/history for billed periods. For contracts on the calendar billing cycle, which is only available for contracts that have digital lines only on them, the calendar contract method will be used, with total line costs (for CPM and Flat Rate lines) averaged over the length of the line for lines that run for multiple months. (This feature is only available on version 8.1 and above.)


Gross/Net/T-Net

  • Gross: uses gross dollars.
  • Net: uses net dollars (gross minus agency commission).
  • T-Net: uses triple net (gross minus agency commission minus acquisition cost)
    • Note: On v71 and above, if the Site Options question “Acquisition Commissionable for Conventional and Selling Vehicles” is checked on, and a “Station Acquisition Commission” percentage is defined for the vehicle and selected date range, then the acquisition cost used will have the acquisition commission percentage deducted from it.
    • If the “Acquisition Cost Only” checkbox on the Billed and Booked report selectivity screen is checked on, the T-Net option will not be available.


Contract: A single contract number can be entered in this field for testing purposes.


Include (Types): choose the types of contracts, Air Time, NTR, etc., that should be included.


Use

  • Pkg: use the package for non-conventional lines instead of the hidden line vehicles. This option is typically not used.
  • Air: use the hidden line vehicles for non-conventional lines. This option is defaulted on.

Additional Options

For standard lines – subtract unresolved missed: When checked on, any revenue from unresolved missed spots from standard lines will be subtracted from the revenue shown. An unresolved missed spot is a spot that remains in the missed state at the time of the running the report. This setting only applies to standard line spots, not to package lines. This checkbox is not available for invoice method A (invoice as ordered, update ordered vehicle).


Count MGs where they aired: When checked on, this option counts revenue from makegood/outside spots where the spots aired, not where they were ordered. This setting applies to both hidden line and conventional line spots. This checkbox is not available for invoice method A (invoice as ordered, update ordered vehicle).


Skip to new page each group – Skips to a new page at the start of each new group when selected.


Slsp Show Splits: When selected, revenue will be split using the salesperson percentage splits defined on the order. Only available with the following sort options: Owner (always checked on along with Sub-totals by Salesperson); Salesperson; and Vehicle.


Effective Pacing Date: A pacing date can be entered so the report will reflect revenue as of the entered effective pacing date. This option is not intended for use with the Calendar (Spots) version of the report because the Cal (Spots) report is run by spot, and there is no history maintained of where spots were. With the pacing version of the report, only contracts are used (not receivables).


Acquisition Cost Only: When checked on, Acquisition Costs will be used instead of Spot Rates.


Include $0 Contracts: When checked on, any zero dollar contracts within the selected date range will be shown on the report. When not checked on, zero dollar contracts will be excluded.

Cal Spots and Digital Lines

The Billed and Booked Cal Spots report includes digital/ad server lines (version 8.1 and above).


For contracts on the broadcast billing cycle, it gathers all ad server lines that are running for at least one day within the report date range and processes each line. For billed standard months, it uses the invoice amount for that standard month, calculates a daily average using the number of days the line runs for in that standard month, then checks the prior standard broadcast month, and if there are calendar month days from the prior month in the next standard broadcast month, it determines how many days are in that period, multiplies it by the daily average, adjusts the prior month, and subtracts that adjustment amount from the month being processed to get an adjusted calendar month total. If the standard month being calculated ends before the calendar month, it uses the daily average from the next month and applies it to the current month to get the calendar month total for that month. It repeats that process for each month that the line runs for.


For unbilled standard months, it starts by determining how much of the line total cost has not been invoiced, then calculates a daily unbilled average, and uses that unbilled daily average to calculate the calendar month totals for the remaining calendar months, based on how many days are in each calendar month for that line.


Once a single digital line has been fully computed for all months like this, it determines which calendar months are actually being requested by the report parameters, and reports the calendar month values for the requested calendar months.


If the Calendar billing cycle is used, for contracts set to the Calendar billing cycle, each month bills the calendar month amount. The unbilled months are calculated using a daily average for the remaining line cost and the number of calendar days that the line is running for.


Example 1 - unbilled projection:

A digital line has dates 3/27/23-4/2/23 for a contract on the broadcast billing cycle. Broadcast April has not been invoiced yet, so this line is entirely uninvoiced, and it has a total cost of $700. There are 7 days in broadcast April for this line, so it divides the line cost by 7 to get the average of 100. There are five days in calendar March, so it multiplies 5 by the average to get 500 for calendar March to show on the report. For calendar April, it's 2 days times the daily average, for 200 to show on the report.


Example 2 - multi-Calendar month line that has been billed:

A digital line has dates 2/27/23-3/05/23 for a contract on the broadcast billing cycle. Broadcast March has been invoiced for $700. The line runs for 7 days in broadcast March, so it takes the billed amount of $700 and divides it by 7 to get the billed daily average of 100. Broadcast March has 2 days in Calendar February for this line, so to calculate the calendar month February amount, it multiplies the billed daily average by 2, to get 200 for February, which it shows on the report. Broadcast March has 5 days in the March calendar month, so it multiplies the billed daily average by 5 to get 500 for calendar March on the report.
