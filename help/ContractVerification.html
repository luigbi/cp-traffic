Contract Verification

The Contract Verification Screen allows designated users to track and alter a contract’s status throughout the agency verification process.


To use the Contract Verification feature, the screen must be activated and a user must be given rights to access the Contract Verification Screen. Only those contracts entered after the feature’s activation will be processed, all contracts in the system prior to the feature being turned on will not appear in any of the Contract Verification screens.

Setup Information

Site Options

You will need a password from support@counterpoint.net to activate this screen in the Site Options -> Contract (Cntr) Screen.

User Options

Users who will be using the Contract Verification screen need their User Options changed to allow them to see and update the Contract Verification menu. Only the Guide user login will be able to activate this user option. 


To allow a specific user access to the Contract Verification menu, the Guide user must go to the User Options screen, select the users they wish to activate, click the "selected fields" radio button, then switch the "Set Contract Verification" option from red to green. 



The Contract Verification option will only appear to those users that have been authorized to access the screen.

Accessing the Contract Verification Screen

To run Contract Verification, click on Task -> Contract Verification, in the upper left corner.


Three options appear within the Contract Verification menu: Not Verified, Sent to Agency, and Verified. 


  • Not Verified- Will display new contracts
  • Sent to Agency- Will display contracts that have been sent to the agency, but have not yet been verified
  • Verified- Lists those contracts that have been received and verified by the agency

Navigating the Screen

All of the data in any of the three screens can be sorted by clicking on the blue header, i.e. sort by Contract Number, Advertiser/Product, etc.


  • Verified is the current status of the contract
  • Verify Date is the date on which the contract’s status was changed, i.e. from Not Sent to Sent to Agency
  • The Contract, Advertiser/Product, Agency, and Gross $’s all come from the contract header
  • Salesperson comes from the contract header. If there are multiple salespeople on a contract, this will show the salesperson entered first.
  • Date Range is the contract’s flight

Contract Statuses 

A contract’s verification status is completely independent of whether the contract has been scheduled, is or has aired, or is billed. The verification status can be changed at any time.


New contracts will appear in the "Not Verified" menu. 


When the contract gets sent to the agency, the user can click in the "Verified" column, and change the status toggle from “Not Sent” to “Sent”, and click Save. The contract will no longer appear in the Contract Verification -> Not Verified Screen, but will now appear in the Contract Verification -> Sent to Agency Screen.  


When a sent contract has been verified, the user can access the contract from the "Sent to Agency" menu, and change the “Sent” toggle to “Verified”, and click Save. This changes the state on the contract to "Verified", and the contract will now appear in the Contract Verification Verified Screen. 

Contract Verification Report

The Contract Verification report will produce a list of contracts and their verification status within a date span. You can determine the date span and types of statuses to include.
