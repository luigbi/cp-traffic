Prophet Media Star Import

Fixed Length, 80 bytes, Filename: SSSSSMMDDYY.003.  All records terminated with c/r l/f.

(YY = year, MM = month, DD = day, SSSSS = unique 5-character station code that must be entered into the Station code on the Vehicle screen of the vehicle being generated)


1-8

Log Date, mm/dd/yy

9-16

Log Time, hh:mm;ss

17-45

Advertiser/Product

46

Asterisk (Counterpoint record indicator)

62-66

Cart # w/o media code

69-70

Status (1=Aired; not 1=Not Aired

71-80

Counterpoint Internal Spot Code


The import file from prophet contains all spots that were originally scheduled to air for a given vehicle and date.  Exact times are updated from this file into the Spot File.  The spot status field is updated to indicate a spot has been time changed, added, or missed.

  • Select menu item Task->Import->Automation.
  • Click on Browse to find the files to select, and click OK
  • User must decide whether to set the spots that have been moved outside their parameters as a makegood or a spot outside its ordered parameters.  “Set Discrepant Post Log Moved Spots as  MG  Outsides”
  • Click Import


An error log, Imptgen.Txt, is created in the Import directory defined in Traffic.ini which records any errors detected during the import.  

  • “Internal spot code missing” – the import file contains a zero for the Counterpoint Spot Code. Spot may have been added into the Prophet system and not exported from Counterpoint.  
  • “Spot missing for internal code XXXXX” – Associated spot for the imported spot code is no longer in the spot file.  Run SSF check.
  • “Import Failed: Missing Station Code for XXXXX” –  enter a station code  in the Vehicle screen


Any spots scheduled for the selected date and vehicle that are not in the import file are automatically set to missed.


Any imported spot with an event status code other than 1 is set to missed.


Print a Log Status report to verify results, in conjunction with the error log.
