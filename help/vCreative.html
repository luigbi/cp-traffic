vCreative

Overview

vCreative PPO (Paperless Production Order) is a web-based continuity management system that manages the workflow of commercials that runs on the air and online, eliminating costly errors and saving hard-earned revenue.


The Salesperson defines a new Proposal with the Copy Script information entered in the Internal Comment field. When approved, Traffic receives an alert to schedule the Order. 


Copy is entered after the order is scheduled. 


Counterpoint automatically sends the new and/or modified copy inventory to vCreative, updates the copy inventory status. Departments can set a status of ‘hold’ to any inventory that they do not want to send to vCreative.

Setup

vCreative.ini

Place the vCreative.ini in the data folder:  x:\csi\prod\data. All values in the INI are provided by vCreative.

Site Options -> Opt tab

Check on the vCreative export. You need a keycode from support@counterpoint.net.

Site Options -> Copy Tab

This is used to determine what date data to retrieve from vCreative.  It is automatically updated.


Vehicle Options -> Genl Tab

Define EDI Call Letters and EDI Band A, F or H.  Counterpoint automatically adds “M” to each so they become AM, FM or HM for the HD’s.  This is for the station name sent to vCreative.

User Setup

  • All salespeople must be defined as users
  • The User must have the salesperson on the contract selected in the Salesperson field
  • The Name on the Report is sent to vCreative as the vCreative salesperson
  • The email address is sent to vCreative for email correspondence.

Creating the Export

Proposal/Contract 

Select the mandatory Business Category.  This is used to define the Ad Type in vCreative.

Internal Comment 

The Internal Comment is where the salesperson can enter all production and script information, which can later be copied into the Copy Inventory screen.

Copy Inventory 

Select Advertiser, and double click [New]  or press the New Inv button to get the Copy Inventory screen.


The Internal Comment from the order will be in the Script Comment where you can edit the field as needed.  If there is more than one script defined, you will have to copy and paste the script into additional Copy Inventory.

Copy Produced Status

The  status can be changed to Not Sent or Hold.  Not Sent will cause the inventory to be resent to vCreative.

  • Not Sent (user can define)
  • Hold (user can define)
  • Sent (user can define)
  • Produced (user can define)

Action Date

This is the date the Copy Inventory was sent to vCreative or the date the produced copy status was retrieved from vCreative.

Interface with vCreative

When Done in Copy is pressed a transmit box displays the send/retrieve status.

  1. New Inventory is transmitted to vCreative.
  2. Inventory whose dates are modified is transmitted to vCreative.
  3. All completed production of the commercials since the last time retrieved will be obtained from vCreative.
  4. The Last Date checked for completed production of commercials will be updated in the Traffic Site Option.
  5. Two text files are created with the results that can be viewed via Accessories -> Messages.
    1. Vcreativelog.txt.
    2. Vcreativeerrors.txt.  


Note: only inventory that is part of a rotation gets sent. Inventory that has been added/created to the inventory screen, but that is not part of a rotation does not get sent.

Copy Changes to retransmit 

Changes that will cause the copy inventory to be resent to vCreative:

  • Adding inventory to a rotation.
  • Changing the start or end dates of a rotation.
  • Using the inventory in another ‘new’ rotation.


The following changes do not cause copy inventory to be resent to vCreative:

  • Change creative title.
  • Change script.
  • Contract only changes: for example, business category or schedule lines changes.

Send Status

The last message is displayed for approximately 2.5 seconds to allow the user time to see the transmission status. The two options are: 


Completed Successfully!

                Or

Completed With (x number of errors) Errors, See Error Log for More Information.  


Note:  vCreative sends a “200” return value logged in the vcreativelog.txt indicating data was successfully transmitted.  If the 200 value is not received, any errors are logged in the vcreativelog.txt and trafficerrors.txt.  

Error Examples:

Error

Indicates

Get Authorization Failed 

Counterpoint could not reach the server or connect with the database.  Check with vCreative to see if they have any indication that Counterpoint tried to send.

User API key failed

The user credentials were not accepted by vCreative.  Check the contract and see who the salesperson is.  Check the User setup in Counterpoint to be sure that the salesperson defined in the contract is in the Salesperson field for the User.

vCreative Get Newly Completed Copy Failed.  No Newly Completed Copy Will be Imported.  Please Refer to vCreativeerrors.txt

Counterpoint was unable to process the copy inventory status.  A detail examination of the vCreativeErrors.txt will be required to determine what detail message was logged from vCreative.  

Could not Interpret vCreative Return.  No New Completed Copy Will be Imported.  Please Refer to vCreativeerrors.txt

Counterpoint was unable to process the copy inventory status.  A detail examination of the vCreativeErrors.txt will be required to determine what detail message was logged from vCreative.  


Note: For copy inventory entered without a contract number, it uses the current user as the salesperson. If the current user is not defined as a salesperson in User Options, it uses the Name on Report and Email Address from User Options for the currently logged in user.

Package Information

On version 7.1 and above, copy rotations can be entered for package vehicles even when the vCreative feature is enabled. (On earlier versions, package vehicles were filtered out from the copy rotation vehicle field.) However, rotations still must be entered for the hidden lines that make up a package. The purpose of entering a rotation for the package is to get the package copy to appear on the invoice. The purpose of entering rotations for the hidden lines is to get the copy transmitted to vCreative. 


When using packages with vCreative, there are two different copy entry procedures for copy rotations for contracts with packages, depending on whether the package and hidden line copy is the same or different. 


Method 1: If the package and hidden line copy is the same, enter a rotation for the package and hidden lines at the same time (as a single rotation) by selecting all the applicable vehicles from the rotation header vehicle dropdown.


Method 2: If the package and hidden line copy is different, enter the package rotation first, save it, then enter the hidden line rotations. This will get the copy on the invoices for the package, and the hidden line specific copy assigned to the hidden lines. (It’s important to do this in the order described, because if the hidden line copy is entered first, then the package copy, the package copy will supersede the hidden line copy, and therefore the package copy will be assigned to the hidden line spots, instead of the hidden line copy.)

Reporting:  Copy Inventory Producer Status

The Copy Inventory by Producer Status will show you the status of your inventory.


Retrieve vCreative Statuses option:  When the report is requested if this option is checked ON Counterpoint will interface with vCreative to update the copy status of the inventory.

Data exchange

There are 4 API calls for the data exchange:

  1. vCreative requires authorization identifier 
  2. Counterpoint automatically sends copy inventory.  It is created in their system as a draft.
  3. Counterpoint sends changes to vCreative which update the draft in vCreative.
  4. Counterpoint queries vCreative as to what has moved from draft to spot and is now complete since mm/dd/yy Last Retrieval Date.

If there’s no live copy script for a piece of copy, the ‘“Production” : {“script”: “”}’ section will be excluded. The ‘“Production” : {“script”: “”}’ section is only included when there is a copy script.
