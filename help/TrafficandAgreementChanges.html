Traffic and Agreement Changes

Affiliate Agreements are based on programming times defined in the Traffic System. Depending on how your pledges are defined, whenever a change is made to programming in Traffic, you may have to redo the agreements. This section will outline scenarios which do and do not require redoing agreements after program changes are made.

When terminating and re-affiliating agreements, you can always use Fast Add and Fast End to create and end multiple agreements at the same time. Fast Add and Fast End are described in a later section of this document.

Programming Changes that Require Agreement Changes

Avail Changes

If Traffic changes any avail times, the number of avails per vehicle, an avail’s length, or the number of units per avail/pod/break, you must terminate existing agreements, and create new agreements to reflect the changes and keep your agreements and logs in sync. 

Library Start Time Changes

A Vehicle’s Library Start Time determines when all avails are fed. If Traffic makes a change to a Vehicle’s Library Start Time, you must terminate existing agreements, and create new agreements to reflect the changes and keep your agreements and logs in sync.

Selling to Airing Link Changes

If there are any changes made to Selling to Airing links, time changes, number of avails, etc., you must terminate existing agreements for any affected airing vehicles, and create new agreements to reflect the changes and keep your agreements and logs in sync.

Changing a Conventional Vehicle to a Selling Vehicle

If Traffic needs to change a Vehicle’s type from Conventional to Selling, you must terminate all existing agreements with the Conventional vehicle, and create new agreements with the new Airing vehicle. If the Airing vehicle is an existing vehicle with increased inventory, you will need to terminate its existing agreements and create new agreements to reflect the inventory changes.

Programming Changes that do not Require Agreement Changes

Avail Name Changes

If Traffic changes an Avail name, for example “Net Avail” is renamed to “Network Avail”, you do not need to make any changes to your agreements. 

Programming Changes to Sports Events Carried Live

If your affiliates are carrying Sports Events live and programming changes are made, you do not need to make any agreement changes, since the agreement will automatically carry all fed avails live.

Changes to Selling but Not Airing Vehicles

If Traffic makes a change to a Selling Vehicle, but does not make any changes to the Airing Vehicle, you do not need to make any changes to the agreement since the agreement only is affiliated with the Airing Vehicle and has no ties to the Selling Vehicle.
