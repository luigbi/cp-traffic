Custom Demos

You can create a new research book with a custom demo, or add a custom demo to an existing book if it is a new research book. Note: The Budget Research screen does not include Custom Demos.


To create custom demo categories, go to Lists -> Custom Demo.

  1. Select [New] from the dropdown.
  2. Enter the Custom Demo in the Name.
  3. Click Save.


Custom Demo Research

To add a custom demo to a research book, go to Lists -> Research and fill in the research book header including the Book Name and Book Date.


You can select the custom demo from the dropdown in the bottom right corner. 



After selecting a custom demo from the dropdown in the lower right, the demo categories will change to the custom demo categories.


Enter the population for the custom demo in the header, and the custom demo research numbers in the grid by one of the four categories (sold daypart, extra daypart, time, or vehicle).

  • Sold Daypart: When dayparts are imported, they are compared to the dayparts in the latest Rate Card.  If they match exactly, they are stored as Sold Dayparts.
    • If using Radar, an extra field “Group #” will appear that is used to indicate socio-economic categories.
  • Extra Daypart:  If the imported dayparts do not match, they are stored as Extra Dayparts
    • There is no limit to the number of dayparts allowed in a vehicle.
    • If using Radar, an extra field “Group #” will appear that is used to indicate socio-economic categories.
  • Time: When quarter hour data is imported, it is stored as Time data.
  • Vehicle: When Exact Time data is imported, they are stored as Vehicle data, since they pertains to the vehicle as a whole, and refer to no particular daypart.


Vehicle data will always be in a book by itself, where Sold dayparts, Extra dayparts, and Time records could all be in the same Book, if each type exists.

Where Custom Demos are Used

Advertiser Screen

Custom Demos can be defined as default demos for advertisers, which are used as the default values when entering proposal/contracts. (Custom demos sort to the bottom of the demo list.)


Orders and Proposal 

All demos in the system show in the Demos dropdown in the Proposal and Orders Screen Header. The advertiser default values are use if defined as pre-selected values. Up to four demos can be selected.

Speed Bar 

The demos specified in the contract header are displayed in the Speed Bar header. The selected demo is used to compute the values shown in the speed bar grid.

 

Demo Bar

The demos specified in the contract header are displayed. The first one in the header is pre-selected. The selected demo is used to compute the values shown in the demo bar grid.

Proposal XML

If there are custom demos on a Proposal, such as Hispanic Men and Women, they will be included with the standard demo and with a comment indicating ethnicity.


Four demos if defined will be sent:


Comment Codes: HW, HM, HA (Hispanic Women, Men, All), BW, BM, BA (Black Women, Men, All), UW, UM, UA (Urban Women, Men, All), SW, SM, SA (Spanish Women, Men, All)


The following Proposal XML group names are supported: Adults (A), Men (M), Women (W), Teens (T), Children (C), Persons (P), and Homes (H).
