Invoice Register Export

This export includes billing information consisting of Invoice (IN) and Invoice Adjustments (AN) transactions, which will be gathered using the transaction date against the dates entered. It includes Air Time and NTR, Cash, Trade, Political, and Non-Political types. Any span of dates can be used to extract data from both the History and Receivables files. The only selection is Transaction Start and End Dates.


This export can be run on demand using any date span from the Task -> Export -> Invoice Register menu.


The output file path uses the standard Counterpoint export folder defined in the Traffic.ini.


Invoice Register filename example: 

Invoice Register mmddyy-mmddyy cccccc.csv 


The words “Invoice Register” is hard coded, mmddyy is the start date entered, mmddyy is the end date entered, and cccccc is the Client Abbreviation Name. Exports generated multiple times on the same day will append a letter, starting with “A”, to the filename.

File Format Specifications

  • Information is stored into comma-separated variable files.
  • All alpha fields are in double quotes.
  • Dollar amounts contain decimal points only if pennies exist. If dollars exceed 999.99, commas are omitted.
  • The data is unsorted.
  • One line per transaction is created.
  • Line 1 of the export contains the field heading names.
  • Balancing totals appear on the last line(s).

Additional Fields

On version 8.1 and above, the following additional fields are included: the five vehicle group categories (sub-company, subtotals, market, research, and format); the advertiser reference ID; the agency reference ID; the salesperson internal ID number, and the station salesperson code.

Summary Version

On version 8.1 and above, a summary version of the Invoice Register export can be run by checking on the "Summary Version" checkbox. There are two types of summary versions available: 22 column version, and the Sage Intacct version.

22 column version

The 22 column version creates a summary version of the export that has one row per invoice/transaction type. Because this is a summary version, the billing vehicle, airing vehicle, and vehicle group columns are suppressed.

Sage Intacct version

The Sage Intacct version is compatible with Sage Intacct. It consists of one record for each invoice number for the selected date span.


The following fields are included:


INVOICE_NO: Counterpoint invoice number

PO_NO: Counterpoint contract number

CUSTOMER_ID: for a non-direct advertiser, it uses the Agency Ref ID from the Agency List screen. For a direct advertiser, it uses the Advertiser Ref ID from the Advertiser List screen.

CREATED_DATE: invoice date

TOTAL_DUE: net dollars due for invoice. Excludes commas and dollar sign.

TERM_NAME: from Site Options, Export tab, Sage Intacct Export, Terms field.

DESCRIPTION: combines advertiser name and product name, separated by a hyphen

LINE_NO: shows the number 1 for each row.

ACCT_NO: from Site Options, Export tab, Sage Intacct Export, Account # field.

LOCATION_ID: from Site Options, Export tab, Sage Intacct Export, Location ID field.

DEPT_ID: from Site Options, Export tab, Sage Intacct Export, Department ID field.

AMOUNT: same value as “TOTAL_DUE”

ARINVOICEITEM_ARACCOUNT: same as ACCT_NO

ARINVOICEITEM_CUSTOMERID: same as CUSTOMER_ID
