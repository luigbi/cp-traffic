Troubleshooting

Email Addresses

Email addresses can contain domains from three to five characters in length. If addresses are being rejected due to domain length, CSINetUtilities.DLL must be installed on the user’s machine.


The file can manually be installed in the C:\CSI\Components\VXX folder (where VXX = Version Number, for example, V70). This is optimal for clients that have a limited number of users that send emails, and do not want to have to run CSIClientInstall.


The second way is done as follows:

  1. For each user’s machine that needs to be updated, browse to the c:\csi\csiComponentCheckervXX.txt file 
  2. Open the file in a text editor.
  3. Change the MSICSICLIENTINSTALL:VX.X number and set it back .1. 
    1. For example, if they are on v70, the csiComponentCheckerv70.txt will show MSICSICLIENTINSTALL:V7.2. 
    2. Change 7.2 to 7.1 (for this example. The version numbers shown for the other software version will be different.)
  4. Save and close the file.
  5. Run Traffic.exe as an administrator on the system and it will run the csiClientInstall and update the required component file.


Note: for new systems being set up for the first time, this will be handled automatically when the system is first set up and traffic.exe gets run as an administrator for the first time.

Not Able to Send Emails

The CsiNetUtilities.Dll is required to use the internal Counterpoint email program.  Running a version of csiNetUtilities with an incompatible version of Counterpoint can result in the inability to send email from within Counterpoint. To verify the csiNetUtilities version, look in the c:\csi\Components\vx.x folder, right mouse click the utility, and select Properties -> Detail. The current version is 1.6.1.0.
