X-Digital Export Log

Counterpoint logs a record of each spot after it successfully sent to XDS, so that only spots not exported will be included in any re-export. X-Digital returns a status for everything exported, whether successful or not. All successful, warning, and/or error messages, the username(s) running the export, the selected vehicle(s), and stations are recorded in the XDigital_Export_Log_MM_DD_YY file (retrievable from CSI\Prod\Data\Messages). 

Export Log Contents

The Export Log lists the vehicle and station information differently, depending upon what was selected to export. If all stations are selected for a single vehicle, it shows "all stations". If not all stations are selected, it will list the selected stations.


Example 1: For this export, the user Julie selected all stations affiliated with the George Carlin vehicle to be exported. The Export log lists the username, the vehicle, and ‘All Stations’ instead of listing every station affiliated. 

User: Julie. Exporting:George Carlin and All Stations.


Example 2: For this export, Julie selected two stations affiliated with George Carlin to be exported. The Export log lists the username, the vehicle and the individual vehicles included. 

User: Julie. Exporting:George Carlin and these Stations: KAAA-FM,KBBB-FM


Example 3: For this export, Julie included every vehicle and every affiliate. The Export Log lists the username, and simply says ‘All’ instead of listing every vehicle and affiliate.

User: Julie. Exporting:All


Example 4: For this export, Julie selected two vehicles and all of their affiliates. The Export Log lists the username, and the two vehicles included in the export. Since all affiliates were included, they are not listed on the log.

User: Julie. Exporting:Billy Crystal,George Carlin

Results

You can view the contents through the Affiliate Messages Viewer, by click Accessories -> Viewer (upper left corner), and selecting X-Digital Logs and the date. 


Any warning or error messages can be viewed in the X-Digital Export Log or the X-Digital Export Results text files, located in the CSI\Prod\Data\Messages folder. The export log will also indicate whether the export was real or a test, as well as the station and/or agreement information or program information that was sent.

 

During the export process, any XDigital_Export_Logs found that are over 30 days old will be deleted.

X-Digital Warning and Error Messages

If something was not successfully exported, X-Digital will either return a warning message, or an error message. All warning and error messages are recorded in the XDigitalExportLog file (CSI\Prod\Data\Messages). 


If there is a timeout issue with the export, Counterpoint will attempt to resend three times (the system defaults to trying three times, but the number of attempts can be altered in the XML.INI file). If successful, a message will appear in the results box saying "Error in sending set insertions corrected.  Export OK and continuing." If unsuccessful, the message will read, “XML error in set insertions.  See XDigitalExportLog.txt. Export Halted." 


The file will contain a Program Code for every entry, whether the item was processed, ignored, or terminated, the Agreement Number, and whether the authorization was valid or invalid.  


XDS Program Codes are 0-6:

 0 = Successfully Exported

-1 = Syntax error 

-2 = Range Error 

-3 = Record Not Found 

-4 = Unsupported 

-5 = Internal Exception 

-6 = Authentication Error 

Warning Messages

If an entry has a code of -1, -2, or -3, it is considered a warning. Warning messages will not stop the export. Counterpoint will record the warnings in the X-Digital Export log, and continue processing the export. The log can be viewed from Accessories>Messages.


Example:

PROGRAMCODE=EVENT, ITEM IGNORED.</MSG><MSG CODE="-2" ID="98220" NAME="AUTHORIZATIONID">DETECTED INVALID


In the message above, the agreement number ID is the number 98220. This can be used to find the vehicle and station that is problematic. Once resolved and re-exported, the authorization information will be re-sent. 

Error Messages

If an entry has a code of -4, -5, or -6, it is considered an error and the export will be terminated. Counterpoint will record all errors in the X-Digital Export log which can be viewed from Accessories>Messages. This can be used to find the vehicle and station that is problematic. Once resolved and re-exported, the authorization information will be re-sent.

X-Digital Ignored Errors Log

During the export, should any warnings occur, Counterpoint will record them in the X-Digital Ignored Errors log, and continue processing the export. The log can be found in the CSI\Prod\Data\Messages folder.
