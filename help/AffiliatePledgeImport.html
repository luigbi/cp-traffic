Affiliate Pledge Import

The affiliate pledge import will import pledge information for agreements that have been previously added to the Affiliate system but with the pledges not previously defined.

Import Rules

  • There must be Live agreements defined in the system
  • Agreements added via Fast Add must be Live
  • No pledge information can be entered into the Pledge tab prior to the initial import
    • If pledge information exists for the agreement in Counterpoint, pledge records in the CSV will be ignored

CSV File Parameters 

  • The CSV must be formatted without spaces between the comma and the next value
  • Time fields can be in HH:MM:SS format, or HH:MM PM/AM, for example  7:24:30 AM or 7:24 AM
  • This includes Start and End Feed and Pledge times 
  • Feed times and pledge times must be provided in the station’s local time
  • All information pertaining to a single agreement must be sorted together 
  • A single agreement's information cannot be interlaced within the CSV
  • Pledges must account for the entirety of a program feed
  • A single avail within a pledge that spans multiple days can have multiple statuses
  • For example, there could be a vehicle with a 60 second break at 9:16am on Monday through Sunday. On the pledge import for a station, the 9am break is set to "Air Live" on Monday through Friday, and "Not Carried" on Saturday and Sunday
  • Each status types must be its own line in the CSV import


"KAAA-FM",147,"ABC",23,114,1,0,0,0,0,0,1,0,09:16:00,09:17:00,9,0,0,0,0,0,1,0,09:16:00,09:17:00,,,,,,,,,,,,,,,,,,,,, 

"KAAA-FM",147,"ABC",23,114,1,1,1,1,1,1,0,0,09:16:00,09:17:00,1,1,1,1,1,1,0,0,09:16:00,09:17:00,,,,,,,,,,,,,,,,,,,,, 


Every Feed time must have a corresponding Pledge time


If a station does not carry the entire program, any hours/avails not carried will be included in the CSV with the designation of “Not Carried”


The following codes in the CSV file must match those codes in Counterpoint for:

  • Agreements
  • Vehicles
  • Station IDs

Imported Conflicts

  • If the Call Letters do not match between the CSV and Counterpoint but the Vehicle, Agreement, and Station ID codes match, the agreement pledge will be updated
  • If the vehicle name does not match between the CSV and Counterpoint but Vehicle, Agreement, and Station ID codes, the agreement pledge will be updated

Ignored Items

  • If the Agreement, Vehicle, or Station ID codes in the CSV file do not match those codes in Counterpoint, those records will be ignored
  • If pledge information exists for the agreement in Counterpoint, pledge records in the CSV will be ignored
  • If no avails are located within the feed times defined in the CSV, the pledge will be ignored 
  • If a feed day within the CSV conflicts with the feed days defined in Counterpoint, the pledge will be ignored 
  • All pledge records for a single agreement will be ignored if any one of the records contains an invalid feed time or day
  • Any duplicate information within the file will be bypassed


A message recording all conflicts and ignored results will be written to the ImptPldg.CSV and saved in the CSI\Prod\Data\Messages folder.

File Fields

The pledge import will recognize the fields below, provided in order, without spaces between the comma and the next field value. 


Call Letters and Vehicle Names must be enclosed in double quotes, for example, "KAAA-FM" and "Billy Crystal." It does not matter whether the other fields are in quotes.

Place the CSV file in the CSI\Prod\Import folder.

Importing the File

To run the import, go to File -> Import -> Agreement Data -> Pledge Times.


Browse to the import file (CSI\Prod\Import folder), and select the file to import, then click Import.

Import Errors

Any errors or conflicts encountered during the import process will be written to a log file "ImptPldg.CSV" in the CSI\Prod\Data\Messages folder. The file will list the line numbers that contain the errors/conflicts. 


During the import, the system verifies that the internal Agreement ID and the Station ID match. If they do not, a message saying “Agreement and Station not Matching” will appear in the Results window, and the pledge data will not be imported for that agreement.
