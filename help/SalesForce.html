SalesForce

SalesForce is a scalable, secure, mobile, and collaborative cloud based CRM application that tracks all projected sales revenue. The Counterpoint Sales Force interface is a standalone program that runs in the background to export your past revenue and future spot and NTR projected revenue manually or automatically on a nightly basis. 

Upon the initial manual generation, the program will gather information from the earliest spot date entered to the farthest spot ordered in the future, or an end date of your choice, and export the information to Sales Force. Then, every night all unbilled spot information from today’s date forward will be exported. 

What Gets Exported

  • Upon the initial manual generation, all spot and NTR projected revenue is sent from an entered spot date to the furthest ordered future spot date. Each generation after will export all spot and NTR information from today’s date forward.
  • Information associated with spots and NTR items, such as the advertiser, agency, salespeople, sales office, and competitive codes are extracted and sent in a separate Contract file. All records in the Advertiser, Agency, Product Protection, Vehicle, Salesperson, Sales Source, and Sales Office files. 
  • Trade Orders will either show 100% trade, or the appropriate cash/trade percentage split. All cash orders will show 0 for trade.
  • The total gross amounts for each of 3 salesperson splits will be calculated.
  • PSA/Promo contracts are not sent.

Setup Information

The program can be set up to run automatically by editing the Windows Task Scheduler, or manually by setting up a shortcut. The first generation must be run manually, allowing you to select the earliest date to export.

Site Options -> Options

Activate Sales Force in the Site Options -> Options screen. You will need a keycode from support@counterpoint.net.

Traffic.INI

Sales Force has option to export the files to a network ftp location. By editing the FTP Info section in the Traffic.INI, the Sales Force export files can be automatically to an offsite location. If this section is not filled out, the files must manually be sent to Sales Force.

 

The entry must contain the following lines:

[FTP_INFO]

FTPIsOn =1

FTPAddress =XXX.XX.XX.XX

FTPPort =XX (User defined)

FTPImportDir= User FTP import directory.   This is the location where files will be uploaded.   If omitted, defaults to ftp root.

LocalExports=X:CSI\Prod\Export This is the local export folder where the Sales Force files have been placed.

FTPUID = User’s ID

FTPPWD = User’s password to location


Example:


[FTP_INFO]

FTPIsOn =1 (1= in use, 2=not in use)

FTPAddress =xxx.xx.xx.xx (user defined)

FTPPort = xx (user defined) 

FTPImportDir=\SalesForce\Import (This is the location where files will be uploaded. If omitted, defaults to ftp root) 

LocalExports=X:\CSI\Prod\export (This is the local export folder where the Sales Force  files have been placed)

FTPUID = users ID

FTPPWD = password to location


Errors that occur during the ftp processing are logged in the FTPLog.txt, located in the CSI\Prod\data\messages folder.

Automatic Setup

Using Windows Task Scheduler, you can set the program to automatically export projected revenue information at a specific time daily. 


If exporting automatically, you cannot control the date span of the export. The start date will be based on today’s date. The end date will be the date of the spot farthest in the future. The destination for the files will be determined in the Traffic.INI file located in the data folder.

Manual Setup

The manual method is intended to be used the first time the export is run, to extract past data.


The Target field must point to the ExportSalesForce.Exe, found in the CSI\Prod\EXE folder. Adding /UserInput will allow the user to alter the start date for the export.


The Start In must point to the folder in which the Traffic.INI resides, typically CSI\Prod\Data.


Manually running the export allows you to control the date span that is collected and exported. The end date will default to the date of the spot farthest in the future, but can be overridden to a date of your choice.


The program will extract all contracts, however a specific contract to extract can be entered.


Once all information is entered, click Export.


The export files will be stored to the CSI\Prod\Exports folder.

Naming Structure

The name of the file is “SFORCE-Type of File MMDDYY-MMDDYY Client Name.TXT.” 


Example: SFORCE-Spots 040114-122814B Comp.txt

  • SFORCE is hard-coded.
  • Type of File is the name of one of the nine files produced, i.e. Spots, Contract, Advertiser, etc. 
  • MMDDYY-MMDDYY are the start and end dates generated.
  • Client Name is the abbreviated Counterpoint client (from Site Options).


A file generated more than once per period will have a letter appended to its date.  

Example:  SFORCE-Spots 040114-122814A Comp.txt.

File Structure

When the export is generated, 9 files are created: Spot, Advertiser, Contract, Product Protection, Agency, Vehicle, Salesperson, Sales Source, and Sales Office.

Data Changes

Changing Spot Prices in the Past

Any spot’s price can be changed prior to billing. Once billed, the only way to alter a price is to unbill the contract, alter the price(s), and rebill the contract. This can only be done to the most recently billed month. If you need to unbill and rebill a contract(s), you must run the Sales force export manually to ensure all price changes are updated. 

Changes to Sales Personnel 

Should a salesperson leave your network, you must assign all pending/current contracts to another salesperson(s) in the company. Frequently, networks will end the original contract as of the person’s last day, and begin a new contract that references the new salesperson(s). All future Sales Force exports will then reference the new salesperson(s). Past exported spots will not be affected. 


This will show which salespeople got commission on each spot.  If you want prior spot commissions to show the new salesperson, it will be necessary to generate a complete file manually.


If a Salespeople moves to a different sales office or sales source a new salesperson should be put on the account, and the situation becomes the same as a salesperson change.

Vehicle Changes

A Vehicle cannot be changed on a contract schedule line.  Only a vehicle’s scheduled end date can be altered, and only as far back as the most recently billed spot.

Agency Changes

If an advertiser changes its agency, the agency name on the contract must be changed. You can either terminate the original contract as of the change date and start a new contract referencing the new agency, or simply change the agency on the existing contract. 


In either case, all exports of unbilled spots generated after that change is made will show the new agency. Previously billed spots will not be affected.  


This method will show which agency was responsible for each spot. If all prior spots are to show the new agency, you must generate a complete file manually.

Advertiser Changes

If an advertiser’s name was spelled incorrectly, the spelling can be corrected without creating a new advertiser.  


If an advertiser was entered into the system twice with slightly different spellings, Counterpoint has a merge operation to blend the two advertisers into one.  The correction can be made manually in the Sales Force system, or a complete manual re-export of all spots can be generated.
