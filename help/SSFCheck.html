SSF Check

The SSF Check/Fix Screen is a tool that compares the spot summary file (SSF) with the spot detail file (SDF), and looks for discrepancies. When it finds them, it will log the vehicle, date and type of error found. A SSF.txt file is stored in the messages directory anytime you run SSF Check or Fix. 


SSF Check is found in Traffic -> Accessories -> Utilities -> SSF Check.


  1. Run the program in “Check” mode by clicking the “Check” radio button.
  2. Start Date and End Date: Enter the dates to check.
  3. Vehicle List: Select the vehicles to check or click “All vehicles”.
  4. Click the Check button to run the utility.


Each vehicle that has no errors will show “0 Will be Missed”.


When you see “Processing Done” press Cancel to exit SSF Check. 


If any errors are detected, they are listed in the white box. If any errors are detected, contact Support@Counterpoint.net. 

Potential Errors

Running the program in ‘fix’ mode normally corrects any problems. Depending on 

the type of error, when you ‘fix’ the error it may cause spots to become missed, or 

in the case of a fill spot, it may be deleted. 

SSF Errors

  1. Advertiser name error: When the advertiser name has been changed and there is a difference between the advertiser name in the two different files (SSF & SDF). Running fix on this problem corrects the advertiser name. 
  2. Spot length error: When the spot lengths don’t match. Perhaps the spot length was changed on the contract, but for some reason it didn’t schedule cleanly, and a sixty second spot was left when it should’ve been changed to a thirty. Running fix may cause the spot to be bumped. 
  3. Avail error: Too many spots have been booked into a break than the break is defined to hold, or one thirty could be filling up a sixty position. The Spot screen may reflect these errors by showing 2/29 (2 units 29 seconds) remaining when there is only 1/30. Running fix will correct these problems. 
  4. Date missing: When you get this error, run the check again just for the one vehicle, and it may come up with no errors. 
  5. Can’t find SDF: The spot summary file (SSF) is unable to find the corresponding record in the spot detail file. Running fix will correct this problem and delete the spot from the spot summary file. 
  6. Line Contract Code Error: This indicates there is a problem with the scheduled spots, they do not belong to the order. The detailed error information includes the contract, line number, date and time of the scheduled spots and the SDF code. 
  7. Time Disparity. Avail time and spot time do not match.
