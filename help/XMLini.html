XML.ini

The XML.ini is used by X-Digital, Wegener, IDC, and Marketron. 

XML.ini Setup

  1. Create an XML.ini file using the example file that can be found in the x:\csi\prod\setup\SampleINI folder.
  2. Place the XML.ini file in the same folder where the Affiliat.ini is placed.  

X-Digital Single Head End Provider

This is a list of the fields used for X-Digital when there’s a single head end.

[XDigital] 

This is the header section of the single head end XML

Host =

This is the IP address of the server.  X-Digital will supply this information to you.  


⮚  Example:  207.114.205.20

Port =

Port 


⮚  Example:  80

WebServiceURL =

X-Digital will supply the URL of the server and service to you.


⮚  Example:  /svc/XYZdService.asmx

DomainNameURL =

X-Digital supplies this information.  


⮚  Example:  http://xnms.xdigital.com

Security Token =

X-Digital supplies this information.   It is a 128 bit digital ID to prevent unauthorized transmissions to the server.

XML Version =

This is always 1.0 unless otherwise specified by X-Digital.  

XML Encoding =

This is always “utf-8” unless specified otherwise by X-Digital.  


⮚  Example:  utf-8

SoapEnvolope =

This defines the SOAP header that surrounds the XML data. It should always be entered as shown unless specified otherwise by X-Digital.  


⮚  Example:  xmlns:soap='http://schemas.xmlsoap.org/soap/envelope/' xmlns:xsi='http://www.w3.org/2001/XMLSchema-instance' xmlns:xsd='http://www.w3.org/2001/XMLSchema'

SoapBody =

This is always entered as Soap:Body

AuthorizationChunkSize=

This groups 4,000 spot insertion records and authorizations into one ‘soap envelope’, if not changed to a different value in this field. This is optional.

LogFile = 

All error messages are captured in a txt file that is stored in the LogFile location. If the location is not defined, the log file will be stored in the EXE folder. If the location is not valid, the export will continue, but the log file will not be created. 


Change this value to match the system  configuration. Use the mapped drive letter.


⮚  Example:  X:\csi\prod\data\messages\XDigitalLog.txt

ClearLog = 

This value should be N, to prevent the debug log from being overwritten. 

AllPorts =

If 0, the station will be not be able to change the port setting. If left out of the INI or greater than 0, all Ports will be available to the station. This is optional.

ReceiverIDSource =

This is an optional setting that determines from where the Site ID is obtained.




A = use the Site ID defined on the agreement 


B = use the agreement Site ID first if it exists, if not, use the Station Site ID 


S = use the Station Site ID only 


N = Don't send the Site ID 




If this value is not defined, on a single head end configuration, method B will be used. If the value is not defined on a dual head end configuration, method N will be used. Typically this setting only needs to be defined for a dual head end configuration where one head end needs to use one method, and the other head end needs to use a different method of determining the Site ID to use.


Note: stations, agreements, spot insertions, and file delivery all are affected by A, B, or S because they all send the site ID. Stations and agreements are affected by the 'N', but spot insertions and file delivery are not.

Retries =

This determines the amount of times the system will attempt to resend the export if there are any issues. If not defined, the system defaults to three. This is optional.

SharedHeadEndID =

If the X-Digital head end is a shared head end, enter the shared head end ID here. For example, SharedHeadEnd = 1. Counterpoint support maintains a list of shared head end IDs. Please contact Counterpoint support for additional information on shared head ends. If the X-Digital configuration does not use a shared head end, this line should not be added.

Single Head End Example

[XDigital]

Host = 207.114.205.20

Port = 80

WebServiceURL = /svc/XYZdService.asmx

DomainNameURL = http://xnms.xdigital.com

SecurityToken = 9452831D-C072-4E1C-8E23-689A491156BC

XMLVersion = 1.0

XMLEncoding = utf-8

SoapEnvolope = xmlns:soap='http://schemas.xmlsoap.org/soap/envelope/'

AuthorizationChunkSize = 4000

xmlns:xsi='http://www.w3.org/2001/XMLSchema-instance' xmlns:xsd='http://www.w3.org/2001/XMLSchema'

SoapBody = soap:Body

LogFile = D:\csi\v60\Data\Messages\XDigitalLog.txt

ClearLog = N

AllPorts = 0

Retries = 3

X-Digital Dual Head End XML Sections

BackOffice

This section is defined for game agreements and used to send station information

Vantive

This section is defined for non-sports agreements

XDigital-CU

This section is defined for the use of spot insertions and programs (non-sports only) 

X-Digital-CL

This section is defined for the use of spot insertions and file delivery

Wegener

The XML.ini file is where the Wegener definitions are set up. All definitions except for the “Export =” should be provided by Wegener. The Wegener export will go to a new folder, CSI\Prod\XML Exports\Wegener XML Exports. This allows you to keep multiple exports separate.


Place the XML.ini file that Counterpoint has given you into the Counterpoint Data directory (CSI\PROD\DATA).

Section Definitions

[Wegener]

Application Name

XMLVersion = 

Defined and provided by Wegener  

  • Example: 1

XMLEncoding  =

Defined and provided by Wegener  

  • Example: utf-8

Playlist_Schedule  = 

Defined and provided by Wegener  

  • Example:xmlns:msdata=http://www.../xml/g xsi:SchemaLocation=“http://www..../xml/g” Rx_Group_Membership=xmlns=http://www..../xml/gr xmlns:xsi=http://www.... xsi:schemaLocation=”http://www.../gr”

MP2FilePath  = 

Defined and provided by Wegener  

  • Example: /Assets/Spots

Export = 

This is where Counterpoint stores the Wegener export.

  • Example: \csi\prod\XML Exports\Wegner XML Exports


Wegener Example

[Wegener]

XMLVersion = 1.0

XMLEncoding = utf-8

ClearLog =

UseSecure =

True means to use https:// instead of http:// when sending.  This is not case sensitive.

UseOldMethod  =

True means to use csiUtils.dll  that way, if something goes wrong with the new method, the system can 'roll back'.  This is not case sensitive.

UseSecureProxy =

True means to use https:// instead of http:// for the proxy address. This is not case sensitive.

 ProxyTestUrl =

This sets set the proxy and tests that it works by going to the entered URL, i.e.  www.Counterpoint.net.

IDC XML.ini Definitions

[IDC]

Section Name

URL = 

This is the URL provided by IDC.

  • Example: http://XXX.XXX.XX.XXX/PMWebService/PMWSDL.asmx

Backup =  

IDC can switch to a backup URL if the main URL goes down for any reason.  To switch URLs, a second URL backup field line must be added to the XML.INI. If the second line is not entered, the export will still function, it just will not run the backup URL system.

  • Example: http:// XXX.XXX.XX.XXX /PMWebService/PMWSDL.asmx

IDC Example XML INI Section

[IDC]

URL = http://XXX.XXX.XX.XXX/PMWebService/PMWSDL.asmx 

Backup =  http:// XXX.XXX.XX.XXX /PMWebService/PMWSDL.asmx
