Interface Options

The Interface Options are used for the Commercial Summary to determine the dayparts. The rest of the options are used for various export files that are created for the vehicle.



  • Commercial Summary Report Information – This defines the dayparts for this vehicle that are used for closing reports.  
  • Program Code Daypart Mapping – This is used to move specific avails from one daypart to another daypart within the same time zone on the Commercial Summary Count.
  • Show Cart # with Bulk Feed Export
  • Yes will show the cart number on the export report.
  • No will suppress the cart number on the export report.
  • Generate Cross Reference Information in Bulk Feed Export
    • Yes will show this vehicle’s commercial information on the Bulk Feed Cross Reference Report 
    • No will exclude this vehicle’s commercial information from the report.
  • In Export of Clearance Spots Treat Vehicle as if Selling Vehicle
    • Yes will assign an “S” indicator on the export report for spots from this vehicle.
    • No will treat this as an “A” airing vehicle.
  • In Export of Clearance Spots Show All Events after 11:59:55PM as 12:00:01A. 
    • Yes will force the start time for all commercials and programs that wrap into the 12AM hour as 12:00:01A.
  • Bulk Feed Group Number – Enter the Bulk Feed group number, or “-“ to indicate exclude from Bulk Feed.  
    • Enter the alpha character to combine vehicles Bulk Feed.  This can be used to group vehicles together for the Bulk Feed.  The rotations for those vehicles assigned the same group number will be combined together or matched up and prevent duplicate rotations from being exported. 
  • ARB Code- Enter the vehicle’s Arbitron code
  • Contract Export Vehicle Extension- Enter the two-digit number that will be appended to the order when exporting the order vehicle information.
  • Radar Code- Enter the vehicle’s Radar Code. This corresponds to the RADAR research import network code and is used to match the vehicle on the research import to the vehicle in Counterpoint.
  • Station Feed Export:  Used only by Cumulus
  • EDAS Time Window Duration.  Enter the seconds.
  • KenCast:  Include Rotation Reference in Envelope.  If selected will include the rotation.
  • Vehicle ID  - Unique two characters to identify the vehicle for the Station Feed Export.
  • Show Cart #’s (on Station Feed)  - If you want the cart numbers to be included in the Copy Feed export.  Normally set to NO.
  • Include in Cross Reference (on Station Feed) – Normally yes.
  • The Airwave ID is obtained from Airwave, and is only necessary if using that export.
  • Export ISCI by Pledge: This is used with the Mr. Master/ISCI export to determine where the program breaks are. Each program break is indicated with an “E” command in the export. When unchecked, the Feed start times will be used. When checked, the Pledge start times will be used to determine where the program breaks are. 


X-Digital Options 


There are three delivery methods for X-Digital; Hour and Break (HB) or Hour and Break and Position (HBP) and/or ISCI. In versions prior to V7.0, you may only select one method.


Counterpoint assigns the code to each vehicle/program, and then, numbers the hours, breaks, and positions within the break. The Counterpoint Setup is defined as Hour/Break or Hour/Break/Position. 

Hour/Break Method 

Avail Form H#B#: For each day on each station, Counterpoint creates the XDS data file that contains the codes for each program, hour, and break, where a regional spot is booked, along with the ISCI code for that regional spot. 


When the export is created, all spots in the break will be sent if one spot has regional copy.

Hour/Break/Position Method 

Avail Form H#B#P#: For each day on each station, Counterpoint creates the XDS data file that contains the codes for each program, hour, break and position, where a regional spot is booked, along with the ISCI code for that regional spot.


When the export is created the regional copy is sent.


  • The Program ID field is used with the HB and HBP Avail forms. It can be up to eight alphanumeric characters and is case sensitive. It must match the case of the code as it was specified in the X-Digital when it was setup. If they do not match, it will not work properly. 
    • As part of a special feature, the word “Event” can be entered as the Program ID for sports vehicles, which allows unique program IDs to be defined on the programming screen for the different events.
  • ISCI Prefix: The ISCI Prefix appears prior to each ISCI name for the ISCI Cross Reference Export.
  • Delivery Save: is where the ISCI will be saved on the Affiliates X-Digital PC
  • CF: Compact Flash
  • HDD: Hard drive
  • NAS: Network Attached Storage

Merging Spots between Programs

A special feature is available with the “by break” avail form (HB and HBP) that allows spots to be merged from one vehicle into another vehicle that air at the same time on the X-Digital export. To use this feature, enter MERGE in the Program ID field for the vehicles that the merge spots will be coming from. 



If the vehicle that the spots are coming from is part of a selling/airing group, enter the word MERGE in the airing vehicle Program ID field.


When using this X-Digital merge feature, check the checkbox “Honor Program ID Merge” for the vehicles that the spots will be merging into.



When setting up this feature, the breaks in the Merged vehicles must be encompassed by the program start and end times of the vehicles set to "Honor Program ID Merge". Be sure to review the avail times for the merged vehicles to ensure that they are within the start and end times of the program libraries of the vehicles they are being merged into. (Program library times can be viewed on the Programming screen and on the Agreements screen.)


Once this feature has been set up, when the X-Digital export is generated, the Merged vehicles will no longer appear in the X-Digital export vehicle list, and the spots from the Merged vehicles will be merged into the corresponding "Honor Program ID Merge" vehicles. The breaks on the Spot Insertions export will be renumbered accordingly. 

National ISCI Model Interface (ISCI)

The National ISCI Model Interface uses Cumulus satellite services.  For each day for each station, the Counterpoint file for XDS contains the ISCI code for the generic spot being replaced, and the ISCI code for the regional spot that replaces it. 


  • Vehice ID: The Vehicle ID field is used with the ISCI Avail form. It can be up to three characters, and must be greater than 0.
  • ISCI Prefix: The ISCI prefix appears prior to each ISCI name for the ISCI Cross Reference Export.
Airing Instructions for spots: 

Replacement ISCI information in the file is sent only once for a day. If the national generic spot runs 10 times in a day all spots will be replaced. This method lessens the volume of ISCI data, but prevents multiple regional spots from rotating in place of one generic spot. This is not a Counterpoint limitation but a design specification.
