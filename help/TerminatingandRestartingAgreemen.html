Terminating and Restarting Agreements

There are several scenarios in which you may have to terminate, reactivate, or retroactively start an affiliate agreement. 

Terminating an Affiliate Agreement

There are three places in which you can enter an agreement’s termination date.

The Last Date On Air represents the last date of the affiliate agreement, and the last date for which an affiliate will receive a log. The Last Date On Air must be a Sunday date.  

If the Site Options setting “Allow Agreement Start, End and Drop Dates” is set to “Yes”, the following additional date fields will be available: Start Date, End Date, and Drop Date.

  • The End Date field is used to note the end of the agreement, but does not actually stop affiliate logs from being sent. This date is referenced on certain affiliate reports.
  • The Drop Date field will stop affiliate logs from being sent. Logs will not be sent after the Last Date On Air or the Drop Date have past (whichever is earliest).

Ending an Agreement Retroactively 

There are times when an agreement must be terminated retroactively – in other words, the termination date is already in the past at the time the agreement is being terminated. Common reasons to retroactively end an agreement include: late network notification of program drop, a pledge entered incorrectly, or a change in the delivery/posting method.

When terminating an agreement, any unposted spots on the station website and in Affiliate Affidavits will be removed. If there are spots that were posted for some reason after the intended termination date, the “Unpost and or Delete Spot” utility can be used to clear the spots. See the “Affiliate Utilities” document for more information on this utility.

Terminating an Agreement with Unposted Spots

Enter the last date that the affiliate should have received logs in the Last Date on Air field. The date must be a Sunday, and must be after the last date posted on the Electronic Affidavit System. (If a date prior to that of the last posted spots is entered, you will receive a warning message, and will need to use the “Unpost and or Delete Spot” utility to remove the posted spots before terminating the agreement.)

After entering the Last Date on Air and clicking “Save as Changed”, you will receive a warning message that spots will be deleted.

Once you click OK, you will receive confirmation that the weeks have been deleted for any affiliate spots after the Last Date on Air both on the station website (if applicable) and from the Affiliate Affidavits screen. 

Starting an Agreement Retroactively

Ideally, the network will be alerted that a station is now an affiliate prior to the first date they should begin receiving logs/affidavits. Sometimes, this information is not conveyed, and a new agreement will have to be started with an On Air Date in the past. 


To create a new agreement that starts in the past, begin creating a new agreement for the station and vehicle. Enter the first Monday of the week they are supposed to get logs in the On Air Date. Fill out the pledges and other required fields then save the new agreement. When saving, you will receive a message indicating the week span that will be added (to the Affiliate Affidavit screen). If the station requires web logs for the added weeks in the past, run the CSI Electronic Affidavit export to export the required weeks to the web for the newly added station and vehicle.

Restarting an Agreement After it has Been Terminated

When an agreement that was terminated needs to be started again, bring up the terminated agreement and enter the first Monday of the week they are supposed to get logs in the On Air Date, and delete the date from the Last Date On Air field. Update the Pledges and other fields if required, then save the new agreement.

Creating a Multicast Agreement for a Previously Non-Multicast Station

To create a multicast agreement for a station that was previously not multicast:

  • If an agreement already exists for the station and vehicle that should be multicast, first terminate that agreement.
  • Change the station to be multicast in the Stations screen->Multicast tab, if it’s not already part of the intended multicast group.
  • Create a new multicast agreement for the station and vehicles that require multicast agreements.
  • When creating the agreement, on the Pledge tab, click the other stations in the multicast group that you wish to create multicast agreements for. 
  • Once you create one agreement and Save as New for multicast stations, all the stations selected on the Pledge tab will be multicast.
  • Because all stations in the multicast arrangement will begin to receive logs, it is crucial that all agreements are terminated prior to creating the new multicast agreements.
  • If the start date of the multicast agreement was in the past and multicast logs need to be exported to the web, re-export the logs using the CSI Electronic Affidavit system for the selected vehicle and multicast stations.

Tracking Overdue Affidavits

When agreements are stopped and started retroactively, some weeks that are actually overdue may be flagged as expired depending on how the agreements overlap.

To be sure that all Overdue Affidavits are included when running the Overdue Affidavits report, check the “Include Expired Agreements” option.
