WO Invoice Export

The WO Invoice Export is an export of invoices similar to the invoice register export, but with one record per invoice number, and meant for importing into an external system. The export format is in XLS format, and Microsoft Excel is required for running this export. This export includes air time, NTR, Cash, Trade (optional), Political, and Non-Political types. Invoice Adjustments (AN Transactions) are also optional.

For additional information about enabling this export, please contact Counterpoint.

Setup Instructions

Traffic Site Options

Options Tab

On the Options tab, check on the “WO Invoice Export” option. A keycode is required for enabling this export. 

Export Tab

On the Export tab, once the WO Invoice Export option has been checked on, the following WO Invoice Export options can be entered:


  • Property: for entering the Property value that will be included on the export.
  • Prefix: for entering the contract and invoice number prefix, which will be added as a prefix to the Counterpoint-generated contract and invoice number on the export output.
  • Billing Group: for entering the Billing Group value.

Advertiser List Screen

The Ref ID field stores the Advertiser GUID. When first setting up the export, enter the Advertiser GUID from the external system for the corresponding advertiser into the Ref ID field. This value will also be used by the Custom Revenue Export as the External Advertiser ID. For Direct Advertisers, the AgencyGUID assigned by the external system should be entered in the Direct Ref ID field on the Advertiser List screen.

Agency List Screen

The Ref ID field stores the Agency GUID. When first setting up the export, enter the Agency GUID from the external system for the corresponding agency into the Ref ID field. This value will also be used by the Custom Revenue Export as the External Agency ID.

Salesperson List Screen

On the salesperson list screen, enter the AE code in the station salesperson code field for each salesperson. 

Export Screen

This export can only be run manually. It cannot be run as an automated task. To access the export, select “WO Invoice” from the Traffic Export menu. This will bring up the export screen, as shown in the picture below. 


To run the export, make the selections needed (described below) on the export screen then press the Export button to generate the export.


Start Date/End Date: each invoice with a transaction date within the entered start date/end date range will be included.


Include Trade: by default, trade invoices are excluded. Check this option on to include trade invoices.


Include Adjustments: by default, invoice adjustments are excluded. Check this option on to include invoice adjustments.


File Type: The Excel (XLS) option is selected by default, and when selected, an XLS file will be created (Microsoft Office must be installed on the system running the export for this option to work). The CSV option will generate a CSV file, which can be opened and saved as an XLS file, for example, if Office is not installed on the system being used to run the export.


Browse: used to browse to a non-default save path for the export file. The filename includes the date range and the client name from Site Options. Letters are appended to the export filename if multiple exports are run for the same date range.


Upload to Amazon Web Bucket: the export can be uploaded automatically to an Amazon S3 Web Bucket by checking this option on and filling out the required fields.


BucketName: The name of the Amazon bucket you wish to upload the exported file to.


BucketFolder: If the file should be uploaded to a sub-folder of the bucket, specify it here. The folder name is case sensitive, and the folder name entered must match the actual bucket folder name. If there’s a typo, it will either fail to successfully upload or create a new folder. Spaces, commas, ampersands, and underlines are okay to use in the folder name. Slashes and the vertical bar symbol are not allowed. The BucketFolder parameter is optional. If not specified, the file will be uploaded to the root folder of the bucket.


Region: Enter the AWS Region the Amazon bucket was created in. Supported regions: AFSouth1, APEast1, APNortheast1, APNortheast2, APNortheast3, APSouth1, APSoutheast1, APSoutheast2, CACentral1, CNNorth1, CNNorthWest1, EUCentral1, EUNorth1, EUSouth1, EUWest1, EUWest2, EUWest3, MESouth1, SAEast1, USEast1, USEast2, USGovCloudEast1, USGovCloudWest1, USWest1, USWest2


AccessKey: A long term credential for an IAM user or the AWS account root user, used to authenticate requests to AWS.


PrivateKey: A long term credential for an IAM user or the AWS account root user, used to authenticate requests to AWS.


KeepLocalFile: Can be set to Yes or No. Defaults to No if not set. A setting of Yes will retain the local file in the export path specified. A setting of No will delete the local file after a successful upload to AWS.


Note: when running the export, the export is logged in the Data\Messages folder on a log file named “WOINV.txt”.

Excel Name Manager

When the XLS version of the export is generated, the export process will automatically select all columns and rows with data and create an Excel name manager definition, with the name of “WOImport”, so that the XLS file will import successfully into the external system without requiring manual editing.
