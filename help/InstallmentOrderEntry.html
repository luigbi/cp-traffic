Installment Order Entry

Orders are defined as using installment billing prior to the first billing on the order.  Once a contract has been invoiced, if it was invoiced as an installment order it cannot be changed to a non-installment order; and if it was invoiced as a non-installment order, it cannot be changed to an installment order.

Important information about NTR and agency commissions

When entering a contract that will use the Installment Billing feature, if the advertiser has an agency and is NTR only, the NTR must either all be set to "agency commissionable = yes" or "agency commissionable = no", there cannot be a mix of the two commission settings for a single contract. 


If the advertiser has an agency and there is both NTR and Air Time on the contract, the NTR commission setting must be set to Yes. (If the advertiser is a direct advertiser, then the NTR agency commissionable setting will be set to No automatically. This is true whether there is NTR only or NTR and Air Time on the contract.)

Order Entry Procedure

  1. Enter the complete header and line/flight area for the contract and enter any NTR that is required. 
  2. Select the Installment Tab. Note: the total gross of the contract and the input gross in the contract header must be in balance before installment billing can be allocated on the installment tab. If the installment billing tab is pressed when the contract is out of balance, the user will receive a warning message that reads “Gross amount and total line amount plus NTR not in balance”.
  3. On the Installment tab, change any of the input options to define how you want the installments invoiced.
  4. Starting invoice date:  This is the first month that you would like to begin invoicing the order. The first billable date of the flight will be shown, but you can select any billing date with the flight from the dropdown
  5. # Months:  Enter the number of months you wish to generate invoices. This does not mean you will generate an invoice every month, but you need to enter the total number of months between the generation of the first and last invoice (enter “1” if you are billing up front or at the end of flight).
  6. End Invoice Date: Any date from the first billing date to the last billing month can be selected from the dropdown (the number of months may change depending on the date selected)
  7. # Months on/# Months off: Using the number of months on/off, define the frequency of when your client will be invoiced for installment amounts.
  8. Total Gross: This field is automatically calculated.
  9. Select the ‘Generate’ button to create the billing layout for the pattern defined.

Installment Allocation

When pressing the Generate button, the system will create the amounts ‘to invoice’ based on the months entered in the grid. 


  • The blue columns can be sorted by vehicle or date for flexible input.
  • The Totals, gross amount ordered and billed amount from the ‘to invoice’ column must be in balance. If there is a discrepancy, the totals in the bottom right will be in red and you will not be able to save until the discrepancy has been resolved.
  • Each month between the start and end dates defined shows the ordered amount. You can change the amount dollars in the ‘To Invoice’ column to meet the client billing specifications by manually typing in the amount. 
  • The amount to invoice can be changed until the order is billed for the first time. Once the contract has been invoiced, you cannot change the amounts to invoice in the future.  If you do, the revenue vs. accounts receivables will be out of balance. If you have a need to change the billing amounts, contact  support@counterpoint.net. 
  • Once the billing amounts have been defined and your totals are in balance, save your order.  

Sports Vehicles with Event Revenue and Installment Billing

If a proposal/order has sports vehicles on it and the option “Event Revenue” is checked on in Traffic Site Options -> Options -> System Options, then Installment Billing cannot be used for that contract. In addition, if an order has installment billing already defined and the user attempts to add a sports vehicle to the contract, the user will be prevented from doing so. In both cases, the user will receive a warning message that reads "Sports and Installment disallowed when using Revenue By Events on the same contract".
