Web Vendor Troubleshooting

This section lists different warnings and errors that can occur during operation, with troubleshooting steps on how to resolve them. Users will be notified when there is an issue by the alert system.

Vendor Alerts

Vendor Alerts are shown in the following places:


  • The button "WVM" that is shown in the Title Bar of the Traffic and Affiliate system will be green if there is no issue. It will be yellow when first starting, when it cannot make a connection to the web, or the export is currently running. It will also be yellow until the first successful export. It turns red if there's an alert. 
  • The red “Alert” button that appears at the top of the Affiliate system (near the date and time) will be shown when there's an actual alert to display. Press this Alert button and click the “Web Vendor” radio button to see information about the alert.
  • On the Affiliate Export Center screen, on the Alerts tab, export type alerts are also shown.


There are two types of warnings/errors that can be received: general issues, and issues related to a specific vendor service. For general issues, there will be no vendor listed in the “Vendor” field on the Alert screen. For issues related to a specific vendor service, an abbreviation of the vendor name will be shown. The following abbreviations are used:


  • CB: CBS
  • IH: iHeart
  • MM: Mr. Master
  • NC: Network Connect
  • RC: RCS
  • RT: Radio Traffic.com
  • RW: Radio Workflow
  • ST: Stratus
  • SY: Synchronicity
  • WO: Wide Orbit
  • XD: X-Digital ISCI

General Issues

The following is a list of general issues (shown in the “Reason” field on the alert screen) that are not specific to one of the web vendors. 


  • Cannot connect to web database: There is an issue trying to get alert information.  Something is stopping affiliate from connecting to the web. Contact Counterpoint.
  • Error in the code: Contact Counterpoint. See the web’s “VendorServiceLog” and/or “WebVendorExport” for the specific issue.
  • Export currently running: This is not an issue, just a warning that the export is not completed, so having no alerts does not mean successful export.
  • Exports queued to go out: This is not an issue, just a warning the export is not yet started, so having no alerts does not mean successful export.
  • Manager not running: Turn on csiVendorServiceManager by choosing “Start manager” in CSI Vendor Service utility.
  • Missed last scheduled import: Run the import from CSI Vendor Service utility.
  • Not all exports went out: Contact Counterpoint. Run the export again: Manager will export automatically; or, re-export some spots to web to kick off export; or, run exports from the CSI Vendor Service utility.
  • “Test mode”; not retrieving: This may not be an error. It indicates it is in test mode, so if it’s not meant to be in test mode, it can be taken out of test mode in the CSI Vendor Service Utility. Check the “Import-Download Available Files” checkbox in the utility.
  • “Test mode”; not sending: This may not be an error. It indicates it is in test mode, so if it’s not meant to be in test mode, it can be taken out of test mode in the Vendor Services Utility. Uncheck “Export-Generate Test File” in Export Controller tab on CSI Vendor Service utility.

Vendor Specific Issues

The following is a list of vendor specific issues that can occur. With a vendor specific issue, the vendor abbreviation will be shown on the alert screen. Please contact Counterpoint support for assistance with any issue of this type.

Exports

  • General error: Coding issue.  
  • Problem with vendor: The vendor had an issue with export
  • Result unknown: It’s unknown if the export was successful or not.
  • Unknown issue: An unknown issue. Contact Counterpoint.

Imports

  • Could not read: Opening file to read posting info failed.
  • Could not parse: Issue with the xml in the file.
  • General error: Coding issue.
  • Match failed: Ast code returned was not found, or vendor is not allowed to post to agreement.
  • Multicast issue: Issue in posting to multicast stations.
  • Unknown issue: An unknown issue. Contact Counterpoint.
