RADAR Import

RADAR will supply a set of files that contain your research information. Place the files in the csi\prod\import folder. 

Import Preparation

The Network Code from the RADAR import files corresponds to the RADAR Code that is entered in the Traffic Vehicle Options screen, on the Interface tab, in the RADAR code field.



The RADAR Code in Counterpoint must match the correct Network Code on the RADAR import files for the corresponding vehicle. When entering a new vehicle that will be using the RADAR research import, the RADAR Code should also be entered and saved before performing the first RADAR import.

Importing the Files

From the Task->Import->Research menu, select RADAR.


Click the Browse button and select file 0, the first of the six fives that are included as part of the import.


Press OK to return to the Import screen.


The following is an explanation of each field on the Import RADAR Data screen:


  • From File: The first file being imported is shown here. This is the file that was browsed to in the Import folder.
  • Book Name: The name of the book being imported is obtained from the file. The name can also be entered manually by typing it in.
  • Book Date: This is the date of the book being imported, and must be manually entered. Check previous books to follow the naming convention. Typically, it is the first or last day of the month containing the new research
  • Set as Vehicle Default Rating Book Name: When this option is checked, this new book will become the default research book for the vehicles that are on it so that when new proposals are entered for these vehicles, this book will be used by default.

Enter the Book Date in the Book Date field on the Import RADAR screen, then press the Convert Files button to begin the import process.

Verification

After the import is finished, verify the research was correctly imported in the Lists->Research by finding the imported research book and confirming that the imported research data is appearing.

RADAR File Contents

All Output files are composed of 135 character ASCII records in which character #135 is a New Line Character.


File 0:        Record 1:    Title and date of RADAR study

       Record 2:        Type of data (audiences to all commercials or audiences to commercials within programs)

       Record 3:        File type and title of output file 1.

       Record 4:        File type and title of output file 2.

       Record 5:        File type and title of output file 3, if used.

       Record 6:        File type and title of output file 4, if used.

       Record 7:        File type and title of output file 5, if used.


Files 1 and 2

Data records for the file types/titles given on records 3 and 4 of output file 0.  (Note that file 1 will always contain "population estimate" data.)


Files 3, 4 and 5

Each of these files exists only if the corresponding title record on file 0 is present. These files contain the type of data records described in the title record.
