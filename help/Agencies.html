Agencies

Agencies can be entered into the system in two ways: on the Agency screen, or the Proposals and Orders screen.


To enter a new Agency, click New in the agency dropdown and press Tab.



Name: This is the Agency’s name.


Abbreviation: This abbreviation will automatically be filled in. You can alter the default. This appears on certain reports and has a maximum of 7 characters.


City ID: This is the Agency’s City ID. It is used to help identify the agency and appears on certain reports.


Active/Dormant: An Agency can be set to Active or Dormant at any time by clicking on the Active/Dormant toggle.


% Commission: The default Agency Commission is 15%, but can be changed in this field if needed.


Default Salesperson:  This default Salesperson can be overridden in the Proposal/Order Screen.


1 or 2 Place Rating: If “GRP Calculations by Rating, Aud/GRP (2 places), Audience”  is set to Audience in the Site Options -> Audience tab, this Rating Place option will appear in the Agency screen. The default value is one. If left at one, the ratings will display one decimal place on the proposal screen, demo bar, and specific reports (for example, 1.3). If set to two, two decimal places will be shown (for example, 1.34).  


G/L Agency Code, Station Agency Code: These codes are used to identify the Agency on Accounting exports. (On version 7.0 and prior, the G/L Agency Code is labeled “Rep Agency Code”.)


Credit Approval, Credit Restrictions, Payment Rating, Credit Rating: Access to these options are controlled by User Options.

  • Payment Rating in Advertiser and Agency: This allows a user to change an Advertiser or Agency’s Payment Rating in the Advertiser/Agency Screens.
  • Change Advertiser/Agencies Credit Rating: This allows a user to change an Advertiser/Agency’s Credit Rating in the Advertiser/Agency Screens. Orders can only be scheduled if an Advertiser/Agency has been approved.
  • Credit Restrictions in Advertiser and Agency: This allows a user to change an Advertiser/Agency’s Credit Restrictions in the Advertiser/Agency Screens. Orders cannot be scheduled if an Advertiser/Agency has any credit restrictions.


ISCI on Invoice, Invoice Sort, Package Inv Show: Each of these fields are toggles that can be changed for each Agency.

  • ISCI on Invoice: Yes means the client requires their copy information on the spots on the invoice/affidavit and the spots that are missing copy will show on the Missing ISCI Report.  No means ISCI is not required for the client.
    • If an ISCI begins with “WW_”, and an agency requires those letters to be excluded from their invoice, this setting must be set to either “ask – default yes”  or “ask – default no” in Site Options -> Copy, and “ISCI on Invoice” must be set to “W/O Leader”. 
  • Invoice Sort: You can define a category that groups invoices with the same requirements together during the invoice run, such as Notarized Scripts Required.
  • Package Invoice Show: You can see either airing times or daypart ordered times in the airing column.
  • Ordered daypart show Name, Time, or None: This setting is in Site Options -> Invoice. This setting allows you to define how you want to see the package spots in the Ordered area on Invoice Form #1. Package spots are created at the time the preliminary or final invoice is generated. The options are:
  • Name: Prints the daypart name.
  • Time:  Prints the ordered daypart times or override times.
  • None: Prints the package vehicle name.


Contract Address: This is the address that will appear on the contract and invoice. If the invoice address is different from the mailing address, enter the address to receive the invoice in the Billing Address field.


Default Buyer/Phone # and Extension/Fax #: This is the default buyer for the agency, which can be overridden in the Proposal/Order Screen. This will appear on the contract and invoice.

 

Payables Contact/Phone # and Extension/Fax #: This is the default agency AP contact


Lock Box Name: The Lock Box addresses are defined in Lists -> Lock Box and then associated with an agency/advertiser.  Lock Box is used if the payment from specific agencies/advertisers is to be sent to a different location.


There is also a Site Options -> Invoice setting “Lock Box by Vehicle instead of Payee” that may be needed if your vehicles are stations where the payment will be sent


Ref ID: used on the Custom Revenue Export as the External Agency ID and on the WO Invoice Export as the AgencyGUID (version 7.1 and above only).


CRM ID: stores the agency CRM ID from RAB. Included on the RAB export. (Version 8.1 and above only)


EDI for Contracts: Not currently implemented.


EDI for Invoices: If using electronic invoices, you must check “Using EDI Service for Invoices”  in the Site Options -> Export tab, and set up your service in the Lists -> EDI screen. Then select the EDI service being used from this dropdown. You can also select “PDF Email” from this dropdown to enable this agency to use the PDF email feature, which automatically emails final and reprint invoices to the recipients defined for the agency. To configure an agency to use the PDF Email feature, select “PDF Email” from the “EDI for Invoices” dropdown, double click, then enter the names and email addresses of the email recipients. Up to four personnel records can be entered. After entering the email recipients, press Done on the PDF Email Invoice Personnel screen, then save the agency.


Terms: The default Remit terms are defined in Site Options -> Export.


Air Time Tax: If applicable set up the tax terms in Site Options -> Invoice. (Note: this field is named Commercial Tax on version 8.1 and above.)


Proposal Export: If using the Proposal XML export, you must first activate it in Site Options -> Options, “Proposal XML”, set up the call letters in Site Options -> Export, “EDI: Call Letters”, and set up the email information in Site Options -> E-Mail.

  • The Proposal Export Form options are: 
    • XML: Proposal XML.
    • OMD: Format used just for OMD agency.
    • CSI: Format that when imported into Excel at the Agency, produces a Proposal that looks very close to the Counterpoint printed Proposal.
    • Note: on version 7.1 and above, this setting sets the default export form. On earlier versions, this setting determines which export form is used.
  • Call Letters (Call-L): When the agency is set to use the Proposal XML export form, if call letters are defined in this field, those call letters will replace the call letters (from Traffic Site Options) shown on the Proposal XML export for proposals using that agency (available on v7.1 and above only).
  • Band: If using the Proposal XML export, any band entered in this field will override the band entered in the Site Options -> Exports band field. This allows agencies to to have unique bands.
  • Dates: This setting determines whether Monday-Sunday or Flight Air Dates are used on the Proposal XML export. The two options are “Air Dates” and “M-Sun” (available on v7.1 and above only).


Suppress Net Amount for Trade Invoices: The default value is blank or No. If an agency has the setting “Suppress Net Amount for Trade Invoices” set to Yes, then any trade invoice (from 1% trade to 100% trade) for that agency will show $0 in the “Net Amount” field at the bottom of the invoice. If any invoices are processed in this way, the final page of the invoice run will show “Less Trade” and the amount of trade dollars that were suppressed in this way, and that trade amount will be excluded from the final “Pay this amount” total on the final invoice page.

Additional Information

Any field in yellow throughout the System is automatically calculated and updated. These yellow fields provide an aging and credit snapshot for the Agency.

Merge

This will merge agency profiles and contracts together. It must be activated in the User Options -> Selected Fields Screen

  1. Choose the “name to remove” and the “replacement name”
  2. Click the Move button to place the merge in the white list box
  3. Select the Merge Button
