Filemaker Import

The Filemaker import allows new orders to be imported from Filemaker. When enabled, new orders can only be imported from Filemaker, they cannot be entered manually. Orders can only be revised using the Traffic system.

Setup Information

Traffic Site Options

To enable the Filemaker import feature, a Counterpoint keycode is required. It is enabled by checking on the Filemaker option in Traffic Site Options, on the Option tab, in the System Options section.

Input and Output Folders

The Filemaker import will import orders in XML format from the “input” folder. Successfully imported orders will be moved to the “output” folder. 


The “input” folder should be created using the following path (substitute the drive letter as needed):


z:\csi\prod\orders\input


The “output” folder should be created using the following path (substitute the drive letter as needed):


z:\csi\prod\orders\output

If an order is missing any of the above fields, it will be rejected during the import process. For example, if the Advertiser is listed as AC Delco, but there’s no advertiser matching that name in the Traffic system, it will be rejected. A user must enter that advertiser name into the Traffic system, then attempt the import again.        

Importing Orders

Orders are imported by pressing the Orders button on the Jobs screen, which will bring up a new "Transition" screen, described below. 

Transition Screen

The Transition Screen lists any XML files that are in the Input folder and ready to import. From this screen, you can also access the Orders screen to revise a previously imported Order by either double clicking the Modifications button, or by clicking once on the Modifications button and then by pressing the Continue to Orders button.



Any orders that are ready to be imported are shown under the Modifications button. The filename used should be the contract number followed by the start date of the contract. In the picture above, the XML file for contract 670 is available to be imported.


To begin the import process, click the order you wish to import, then press the “Continue to Orders” button (or simply double click the XML file shown on the Transition screen). Note: only one order can be imported at a time.

Validation Process

During this initial stage of the import process, the system will run a validation process on the XML file. If there is an issue, such as a violation of the specification rules (this could be something like the contract number is already in use, the agency on the XML file is not in the Traffic system, etc.), the user will be notified in several ways. The words “Failed validation” will appear in red after clicking the “Continue to Orders” button, and a warning will be written to the “FileMakerToOrdersIssuesLog_mm-dd-yy.txt” log file, which can either be accessed from the Accessories -> Messages -> Filemaker to Orders Issues, from which you can select the log file for the current day to view the contents, or from the csi\prod\data\messages folder. An example of a warning message is shown below.


::::::WARNING:::::::

2:30:16 PM User: Julie. 670-07122017.xml failed validation: Advertiser Name is not valid. Buyer Name cannot be tested.

:::::::::::::::::::


When there is a validation failure, retrieve the log file, find the cause of the validation error or errors, correct them, then try importing again. In the example above, the advertiser name is listed as not being valid because the advertiser name on the XML file does not match any advertiser names defined on the Traffic system. It also lists that the Buyer Name cannot be checked to see if it’s valid because of the non-matching Advertiser name. To correct this issue, the new advertiser and buyer must be added to the Traffic system, before the XML file will import successfully.

Orders Screen

When a new Order has passed the validation process, it will be shown on the Orders screen, where it can be edited as needed and then saved to complete the import process. Some of the fields on the Orders screen are not included on the XML import, such as the Product Protection code, so if those are required, they must be entered manually on the Orders screen.

Save Process

When the Order has been edited and is ready to be saved on the Traffic system, press the Done button. Note that the Save-Stay and Save-Go buttons will be grayed out and unavailable for a newly imported order like this. (If you don’t wish to save the Order for some reason, you can press the Cancel button to exit from this screen without saving it. When doing this, the XML file will remain on the Transition screen where it can be imported at another time.)


To complete the save process, after pressing Done, click “Yes” on the “save changes” prompt, then press “Done” on the “revision number screen” (edit the revision number if needed) to fully import and save the new order on the Traffic system. This will also move the XML file from the Input folder to the Output folder, so that it will no longer appear in the list of contracts that are available to import, and automatically take you back to the Transition screen where you can either select another import file to import, press Modifications to edit an already imported order, or Cancel to exit to the main Jobs screen.

Additional Information

You can press the Orders button at the top of the Jobs screen (near the Help and Alert menu items) to get into the Orders screen directly, bypassing the Transition screen, for example if you just wanted to revise or schedule existing orders.
