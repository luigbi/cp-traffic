Participant Revenue Setup

Sales Source 

The Sales Source determines how the revenue for a vehicle is updated, such as using accounts receivables in Counterpoint, or whether it goes directly to Payment History or to an export. Each Salesperson is defined as being part of a single Sales Source on the Salesperson List screen, so that the revenue for each contract will be handled correctly based on the defined Salesperson Sales Source. Sales Sources are defined on the Sales Source List screen (shown below).



The Invoice Update field is used to define what happens when invoicing a contract using this Sales Source. There are five options: 


  • Receivables: Will update the amount to A/R where cash can be posted against it.
  • History: Will update the invoice amount directly to payment history.
  • Export + History: Will invoice the amount, automatically create an export and update it to payment history.
  • Export + A/R: Will invoice the amount, automatically create an export and update it into the current A/R.
  • Ask by Vehicle: allows the "Invoice Update" setting to be defined by Vehicle (on the Participant tab).


For more information on setting up Sales Sources, see the Lists Screen help document.

Participants 

Every vehicle in the system must have a participant defined. To define participants, go to Vehicles -> Options ->Participant tab. 


When adding a new vehicle, if the Participants are not defined, when saving, a warning message will prompt the user to define the participants. When adding a new vehicle that is modeled from an existing vehicle, check the Participant tab before saving to ensure it is defined correctly for the new vehicle.

On version 7.0 and earlier

To define a new participant split on version 7.0 and earlier:

  1. Select [New] from the dropdown and press tab. 
  2. Enter the first Monday date of a standard broadcast month that will be used as the start date for this participant definition. 
  3. Define the Sales Source (the group that sells the spots for this vehicle). 
  4. Define the Participant (each partner that will be receiving a percentage of the revenue for this vehicle). 
  5. Determine whether the dollars will update to receivables or go straight to history. 
  6. Define the Revenue split percentage. 
    1. If the vehicle has a revenue split among different participants, enter a line for each participant that will receive a portion of what is sold by each sales source. The revenue percentages for all the participants for a single sales source must add up to 100%. 
    2. If a vehicle is owned 100% by a single sales source and participant, then simply define a single line for that sales source and participant for 100% of the revenue.
  7. Click Save. 

On Version 7.1 and above

To create a new participant split on version 7.1 and above: select New from the dropdown in the upper right corner, select the start date from the calendar (using the start of a standard broadcast month), define the participant information as outlined above, and then press Save.


On all versions, the first participant listed must be the owner of the vehicle. 

Viewing and Modifying Previously Entered Splits

Previously entered participant splits can be viewed by selecting the definition from the dropdown in the upper right corner of the Participant tab. 


As shown in the picture below, one participant split definition was from 12/29/14 to 12/27/15, and the current TFN split began on 12/28/15. Both splits can be selected from the dropdown in the upper right corner of the Participant screen.

To enter a new split definition, simply create a new definition using the new split start date. The system will automatically end date the previous definition when saving the new definition.

Multiple Sales Sources 

It's also possible for there to be multiple Sales Source that sell inventory. When this happens, each Sales Source must be defined on the Participant tab, for each participant that participates in the revenue share for a vehicle. 

Two Sales Sources - two participants 

In the example below, there are two Sales Sources: XYZ and Acme Rep. When the Sales Source is XYZ, the owner, Acme Rep, gets 75% of the revenue, and XYZ Corp gets 25%. When the Sales Source is Acme Rep, the revenue goes directly to payment history for the Acme Rep participant portion, which is 90%, and the remaining 10% goes to receivables for XYZ Corp. 


In the picture above, the Owner “Acme Rep” is also allowed to sell inventory and handles its own collections in an external system for its percentage of the revenue for the contracts that it sells. That is why in the example below the Invoice Update setting is set to History. This allows "XYZ" to see a true picture of what their Accounts Receivables are, with the "Acme Rep" portion excluded from receivables for the contracts where the salesperson is defined with the "Acme Rep" Sales Source.
